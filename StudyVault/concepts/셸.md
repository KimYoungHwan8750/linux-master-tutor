### 오답 메모

**콘셸 (ksh)**
- 문제: David Korn이 개발하여 본셸과 호환되면서 C셸의 기능을 결합한 셸은?
- 내 답: tcsh
- 정답: ksh
- 핵심: ksh=본셸 호환+C셸 결합 (David Korn). tcsh=C셸 개선+자동완성 (Ken Greer).

**C셸 (csh)**
- 문제: Bill Joy가 UC Berkeley에서 개발하였으며, 히스토리와 별칭(alias) 기능을 최초로 도입한 셸은?
- 내 답: ksh
- 정답: csh
- 핵심: csh가 히스토리/별칭을 최초로 도입. ksh는 나중에 그 기능을 결합한 셸.

**/etc/shells 파일**
- 문제: 시스템에서 사용 가능한 셸 목록이 저장된 파일은?
- 내 답: /etc/bashrc
- 정답: /etc/shells
- 핵심: /etc/shells=셸 목록, /etc/bashrc=bash 설정 파일, /etc/profile=로그인 설정.

**chsh 명령어**
- 문제: 사용자의 로그인 셸을 변경하는 전용 명령어는?
- 내 답: 모름
- 정답: chsh
- 핵심: chsh=Change Shell. chsh -s /bin/zsh로 셸 변경. chsh -l로 사용 가능한 셸 목록.

**set 명령어**
- 문제: 셸 변수와 환경변수를 모두 출력하는 명령어는?
- 내 답: printenv
- 정답: set
- 핵심: set=전부(셸+환경), env/printenv=환경변수만.

**export 명령어**
- 문제: 셸 변수를 환경변수로 전환하여 자식 프로세스에서도 사용할 수 있게 하는 명령어는?
- 내 답: source
- 정답: export
- 핵심: export=변수 전환(자식 프로세스 전달), source=파일을 현재 셸에서 실행.

**PS2 보조 프롬프트**
- 문제: 명령이 한 줄에서 완성되지 않아 다음 줄로 이어질 때 표시되는 보조 프롬프트 변수는?
- 내 답: PS1
- 정답: PS2
- 핵심: PS1=기본 프롬프트(명령 입력 대기), PS2=보조 프롬프트(다음 줄 이어질 때, 기본값 >).

**/etc/profile**
- 문제: 모든 사용자에게 적용되며 로그인 시 실행되는 시스템 전역 설정 파일은?
- 내 답: /etc/bashrc
- 정답: /etc/profile
- 핵심: profile=로그인 시, bashrc=bash 실행 시. /etc/=시스템 전체.

**/etc/bashrc**
- 문제: 모든 사용자에게 적용되며 bash 셸이 실행될 때마다 읽히는 시스템 전역 설정 파일은?
- 내 답: ~/.bashrc
- 정답: /etc/bashrc
- 핵심: /etc/bashrc=시스템 전체, ~/.bashrc=개인. 둘 다 bash 실행 시 읽힘.

**~/.bash_profile**
- 문제: 개인 사용자에게만 적용되며 로그인 시 한 번 실행되는 설정 파일은?
- 내 답: ~/.bashrc
- 정답: ~/.bash_profile
- 핵심: bash_profile=로그인 시, bashrc=bash 실행 시.

**~/.bashrc**
- 문제: 개인 사용자에게만 적용되며 bash 셸을 시작할 때마다 실행되는 설정 파일은?
- 내 답: ~/.bash_profile
- 정답: ~/.bashrc
- 핵심: bashrc=bash 실행 시마다, bash_profile=로그인 시 한 번.

**로그인 셸 vs 비로그인 셸**
- 문제: 비로그인 셸(터미널 새 창 열기 등)이 읽는 설정 파일의 순서는?
- 내 답: /etc/profile → ~/.bash_profile
- 정답: ~/.bashrc → /etc/bashrc
- 핵심: 로그인 셸=/etc/profile→~/.bash_profile, 비로그인 셸=~/.bashrc→/etc/bashrc.

**설정 파일 적용 순서**
- 문제: 로그인 셸에서 설정 파일이 적용되는 올바른 순서는?
- 내 답: /etc/bashrc → /etc/profile → ~/.bash_profile → ~/.bashrc
- 정답: /etc/profile → ~/.bash_profile → ~/.bashrc → /etc/bashrc
- 핵심: 시스템 로그인 → 개인 로그인 → 개인 bash → 시스템 bash 순서.

**history 명령어**
- 문제: 직전에 실행한 명령어를 다시 실행하는 히스토리 기호는?
- 내 답: 모름
- 정답: !!
- 핵심: !!=직전 명령어, !n=n번째 명령어, history -c=삭제, history -w=파일 저장.

**/etc/passwd 필드 구조**
- 문제: /etc/passwd 파일에서 7번째(마지막) 필드에 저장되는 정보는?
- 내 답: 모름
- 정답: 로그인 셸
- 핵심: 사용자명:비밀번호:UID:GID:설명:홈디렉토리:로그인셸 (7개 필드).
