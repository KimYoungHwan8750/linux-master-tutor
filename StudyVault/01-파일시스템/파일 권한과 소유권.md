# 파일 권한과 소유권 (★★★)

#linux-master #filesystem #permission

## 한눈에 비교

| 명령어 | 기능 | 핵심 옵션 |
|--------|------|----------|
| ls -l | 파일 권한/소유자 확인 | -a(숨김), -R(재귀), -d(디렉토리) |
| chown | 소유자(+그룹) 변경 | -R(재귀) |
| chgrp | 그룹 소유권만 변경 | -R(재귀) |
| chmod | 접근 허가권 변경 | -R(재귀) |
| umask | 기본 허가권 설정 | -S(문자 표기) |

## ls 명령어 상세

파일/디렉토리의 허가권, 소유권 등을 확인하는 명령어.

출력 형식: `rw-rw-r-- 1 june june 120 2024-09-16 19:10 text`

| 필드 | 의미 |
|------|------|
| rw-rw-r-- | 파일 허가권 (소유자/그룹/기타) |
| 1 | 물리적 파일 연결 개수 (하드링크 수) |
| june (첫번째) | 파일 소유자명 |
| june (두번째) | 파일 소유 그룹명 |
| 120 | 파일 크기 (바이트) |
| 2024-09-16 | 최종 변경 시간 |
| text | 파일명 |

**파일 유형 문자**:
| 문자 | 의미 |
|------|------|
| - | 일반 파일 |
| d | 디렉토리 |
| l | 심볼릭 링크 |
| b | 블록 디바이스 |
| c | 캐릭터 디바이스 |
| p | 네임드 파이프 |
| s | 소켓 |

## 허가권 (Permission) 체계

**숫자(8진수) 표기법**:
| 권한 | 값 | 의미 |
|------|-----|------|
| r | 4 | 읽기 |
| w | 2 | 쓰기 |
| x | 1 | 실행 |

예: `rwxr-xr--` = 754 (소유자 7, 그룹 5, 기타 4)

**문자 표기법**:
| 기호 | 대상 |
|------|------|
| u | 소유자 (user) |
| g | 그룹 (group) |
| o | 기타 (others) |
| a | 모든 사용자 (all) |

연산자: `+`(추가), `-`(제거), `=`(설정)

예: `chmod u+x,g-w,o=r file.txt`

## chown 명령어

파일/디렉토리의 소유자와 그룹을 변경.

```bash
chown [옵션] 소유자[:그룹명] 파일명
chown root:admin file.txt    # 소유자를 root, 그룹을 admin으로
chown -R root /home/june     # 재귀적으로 하위 모두 변경
```

## chgrp 명령어

파일/디렉토리의 **그룹 소유권만** 변경.

```bash
chgrp [옵션] 그룹명 파일명
chgrp june test.txt    # test.txt의 그룹을 june으로
```

## chmod 명령어

파일/디렉토리의 접근 허가권을 변경.

```bash
chmod [옵션] 권한 파일명
chmod 755 script.sh       # rwxr-xr-x
chmod -R 777 /shared      # 하위 모두 rwxrwxrwx
chmod u+x,g+r file.txt    # 문자 방식
```

## umask 명령어

새로 생성되는 파일/디렉토리의 **기본 허가권** 결정.

| 항목 | 기본 최대 권한 | umask 022 적용 |
|------|--------------|---------------|
| 파일 | 666 (rw-rw-rw-) | 666-022 = **644** (rw-r--r--) |
| 디렉토리 | 777 (rwxrwxrwx) | 777-022 = **755** (rwxr-xr-x) |

```bash
umask         # 현재 umask 값 확인
umask 0002    # umask를 0002로 설정
umask -S      # 문자 형식으로 표시 (u=rwx,g=rwx,o=rx)
```

> [!important] 시험 포인트
> 파일의 기본 최대 권한은 **666**(실행권한 없음), 디렉토리는 **777**이다.
> umask 값을 빼면 실제 생성 시 부여되는 권한이 된다.

## 특수 권한

| 특수 권한 | 설정 위치 | 8진수 | 기호 | 설명 |
|-----------|----------|-------|------|------|
| SetUID | 소유자 실행 권한 | 4000 | s (대문자 S=실행권한 없음) | 실행 시 파일 소유자 권한으로 실행 |
| SetGID | 그룹 실행 권한 | 2000 | s | 실행 시 파일 그룹 권한으로 실행 |
| Sticky Bit | 기타 실행 권한 | 1000 | t (대문자 T=실행권한 없음) | 해당 파일의 소유자나 root만 삭제 가능 |

```bash
chmod 4755 file    # SetUID 설정
chmod 2755 dir     # SetGID 설정
chmod 1777 /tmp    # Sticky Bit 설정 (대표 예: /tmp)
chmod u+s file     # SetUID 문자 방식
chmod g+s dir      # SetGID 문자 방식
chmod o+t dir      # Sticky Bit 문자 방식
```

> [!warning] Sticky Bit 주의
> Sticky Bit은 **디렉토리에 부여**되며, 사용자(u), 그룹(g)에는 적용할 수 없다.
> 대표적 예: `/tmp` 디렉토리 (누구나 쓸 수 있지만, 자기 파일만 삭제 가능)
