# 파일 권한과 소유권 (★★★)

| 항목 | 명령어 | 기능 |
|------|--------|------|
| 허가권 변경 | `chmod` | 읽기/쓰기/실행 권한 설정 |
| 소유자 변경 | `chown` | 파일 소유자 변경 |

## 허가권(Permission) 체계

파일 권한은 **소유자(user)**, **그룹(group)**, **기타(others)** 3가지 범주로 나뉜다.

```
-rwxr-xr-- 1 user group 4096 Jan 1 file.txt
 ↑↑↑ ↑↑↑ ↑↑↑
 소유자 그룹 기타
```

- **r**(read) = 4, **w**(write) = 2, **x**(execute) = 1
- 숫자 표기: `rwxr-xr--` = 7(4+2+1) + 5(4+0+1) + 4(4+0+0) = **754**

### chmod 사용법
| 방식 | 예시 | 의미 |
|------|------|------|
| 숫자 | `chmod 755 file` | rwxr-xr-x |
| 기호 | `chmod u+x file` | 소유자에 실행 권한 추가 |
| 기호 | `chmod go-w file` | 그룹/기타에서 쓰기 제거 |

### 소유권 변경
- `chown user:group file` — 소유자와 그룹 동시 변경
- `chown user file` — 소유자만 변경
- `chgrp group file` — 그룹만 변경
- `-R` 옵션: 디렉토리 하위 재귀 적용

## 특수 권한

| 권한 | 숫자 | 적용 대상 | 표시 | 효과 |
|------|------|-----------|------|------|
| **SetUID** | 4000 | 소유자 x 위치 | `s`/`S` | 실행 시 파일 소유자의 UID로 실행 |
| **SetGID** | 2000 | 그룹 x 위치 | `s`/`S` | 실행 시 파일 그룹의 GID로 실행 |
| **Sticky bit** | 1000 | 기타 x 위치 | `t`/`T` | 해당 디렉토리에서 소유자/root만 삭제 가능 |

> [!important] s vs S 구분
> - 소문자 `s` = 기존 실행(x) 권한이 **있는** 상태에서 특수 권한 적용
> - 대문자 `S` = 기존 실행(x) 권한이 **없는** 상태에서 특수 권한 적용

> [!tip] 대표 예시
> - SetUID: `/usr/bin/passwd` (일반 사용자가 비밀번호 변경 가능)
> - Sticky bit: `/tmp` (모든 사용자 쓰기 가능하지만 남의 파일 삭제 불가)

## Exam/Test Patterns (시험 빈출 패턴)
| Scenario/Keyword | Answer |
|-------------------|--------|
| "chmod 4755" | **SetUID** + rwxr-xr-x |
| "소유자만 삭제 가능한 디렉토리" | **Sticky bit** (chmod 1777) |
| "실행 시 파일 소유자 권한" | **SetUID** |
| "rwxr-xr-x 숫자 표현" | **755** |
| "허가권 변경 명령" | **chmod** |
| "소유자 변경 명령" | **chown** |
