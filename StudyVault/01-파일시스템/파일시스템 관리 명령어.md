# 파일시스템 관리 명령어 (★★★)

#linux-master #filesystem #mount

## 한눈에 비교

| 명령어 | 기능 | 핵심 옵션 |
|--------|------|----------|
| mount | 파일시스템 마운트 | -t(타입), -o(옵션), -a(fstab 전체) |
| umount | 파일시스템 언마운트 | -a(전체), -f(강제) |
| eject | 이동식 미디어 꺼내기 | (CD-ROM, USB 등) |
| fdisk | 파티션 생성/삭제/관리 | -l(목록), -d(삭제) |
| mkfs | 파일시스템 생성(포맷) | -t(타입) |
| mke2fs | ext 계열 파일시스템 생성 | -t(타입), -j(저널링) |
| fsck | 파일시스템 무결성 점검 | -y(자동 복구) |
| e2fsck | ext 계열 점검/복구 | -y(자동), -p(자동복구) |
| xfs_repair | XFS 파일시스템 복구 | xfs 전용 |
| du | 디스크 사용량 확인 | -s(요약), -h(읽기쉬움) |
| df | 디스크 여유 공간 확인 | -h(읽기쉬움), -T(타입) |

## mount / umount / eject

### mount 명령어

장치(파티션)를 특정 디렉토리에 **연결(마운트)**하여 접근 가능하게 한다.

```bash
mount [옵션] 장치명 마운트포인트
mount /dev/sdb1 /mnt/usb         # /dev/sdb1을 /mnt/usb에 마운트
mount -t ext4 /dev/sdb1 /mnt/data  # ext4 타입으로 마운트
mount -o ro /dev/sdb1 /mnt/data    # 읽기 전용 마운트
mount -a                           # /etc/fstab의 모든 항목 마운트
mount                              # 현재 마운트된 정보 표시
```

| 옵션 | 설명 |
|------|------|
| -t 타입 | 파일시스템 유형 지정 (ext4, xfs, ntfs, vfat, iso9660, nfs 등) |
| -o 옵션 | 마운트 옵션 지정 |
| -a | /etc/fstab에 정의된 모든 파일시스템 마운트 |
| -r | 읽기 전용 마운트 (= -o ro) |
| -w | 읽기/쓰기 마운트 (= -o rw) |

**-o 세부 옵션**:
| 옵션 | 설명 |
|------|------|
| ro | 읽기 전용 |
| rw | 읽기/쓰기 |
| remount | 이미 마운트된 파일시스템 재마운트 |
| loop | ISO 이미지 파일 마운트 시 사용 |
| noexec | 실행 파일 실행 차단 |
| nosuid | SetUID/SetGID 비활성화 |
| usrquota | 사용자 쿼터 활성화 |
| grpquota | 그룹 쿼터 활성화 |

```bash
# ISO 이미지 마운트
mount -o loop image.iso /mnt/cdrom

# NFS 마운트
mount -t nfs 192.168.1.100:/shared /mnt/nfs

# 재마운트 (fstab 변경 후)
mount -o remount /home
```

### umount 명령어

마운트된 파일시스템을 **해제(언마운트)**한다.

```bash
umount [옵션] 장치명|마운트포인트
umount /mnt/usb           # 마운트 포인트로 해제
umount /dev/sdb1           # 장치명으로 해제
umount -a                  # 모든 파일시스템 해제
umount -f /mnt/nfs         # 강제 해제 (NFS 등에서 사용)
```

> [!warning] 주의
> 해당 디렉토리를 사용 중(현재 작업 디렉토리 등)이면 `device is busy` 에러 발생.
> `fuser -m /mnt/usb` 또는 `lsof /mnt/usb`로 사용 중인 프로세스 확인 가능.

### eject 명령어

CD-ROM, DVD 등 이동식 미디어를 **꺼내는(배출)** 명령어.

```bash
eject              # 기본 CD-ROM 꺼내기
eject /dev/cdrom   # 특정 장치 꺼내기
eject -t           # 트레이 닫기
```

## fdisk 명령어

디스크의 **파티션을 생성, 삭제, 변경**하는 명령어.

```bash
fdisk [옵션] 장치명
fdisk -l              # 모든 디스크의 파티션 목록 표시
fdisk -l /dev/sda     # 특정 디스크의 파티션 목록
fdisk /dev/sdb        # 대화형 파티션 관리 진입
```

| 옵션 | 설명 |
|------|------|
| -l | 파티션 테이블 목록 출력 |
| -d | 파티션 삭제 (대화형 모드 내) |

**fdisk 대화형 모드 내부 명령어**:
| 명령 | 설명 |
|------|------|
| n | 새 파티션 생성 (new) |
| d | 파티션 삭제 (delete) |
| p | 파티션 테이블 출력 (print) |
| t | 파티션 유형 변경 (type) |
| w | 변경 사항 저장 후 종료 (write) |
| q | 저장하지 않고 종료 (quit) |
| l | 알려진 파티션 유형 목록 (list) |

> [!tip] 파티션 유형
> - 주 파티션 (Primary): 최대 4개
> - 확장 파티션 (Extended): 주 파티션 1개를 확장으로 사용
> - 논리 파티션 (Logical): 확장 파티션 내부에 생성, /dev/sda5부터 시작

## mkfs / mke2fs 명령어

### mkfs (Make FileSystem)

파티션에 **파일시스템을 생성(포맷)**한다.

```bash
mkfs [옵션] 장치명
mkfs -t ext4 /dev/sdb1      # ext4로 포맷
mkfs -t xfs /dev/sdb1       # xfs로 포맷
mkfs.ext4 /dev/sdb1         # 동일한 표현
```

| 옵션 | 설명 |
|------|------|
| -t 타입 | 파일시스템 유형 지정 (ext2, ext3, ext4, xfs 등) |
| -c | 배드 블록 검사 후 포맷 |

### mke2fs

**ext 계열** 파일시스템을 생성하는 전용 명령어.

```bash
mke2fs [옵션] 장치명
mke2fs -t ext4 /dev/sdb1     # ext4 생성
mke2fs -j /dev/sdb1          # 저널링이 포함된 ext3 생성
mke2fs -b 4096 /dev/sdb1     # 블록 크기 4096바이트로 지정
```

| 옵션 | 설명 |
|------|------|
| -t 타입 | 파일시스템 유형 (ext2, ext3, ext4) |
| -j | 저널링 추가 (ext3 생성) |
| -b 크기 | 블록 크기 지정 (1024, 2048, 4096) |
| -i 크기 | inode당 바이트 수 지정 |
| -c | 배드 블록 검사 |

## fsck / e2fsck / xfs_repair

### fsck (File System Check)

파일시스템의 **무결성을 점검하고 복구**한다.

```bash
fsck [옵션] 장치명
fsck /dev/sdb1             # 파일시스템 점검
fsck -t ext4 /dev/sdb1     # ext4 타입으로 점검
fsck -y /dev/sdb1          # 모든 질문에 yes 자동 응답
```

| 옵션 | 설명 |
|------|------|
| -t 타입 | 파일시스템 유형 지정 |
| -y | 모든 질문에 자동으로 yes 응답 |
| -n | 점검만 하고 수정하지 않음 |
| -a | 자동 복구 (비대화형) |

> [!warning] 주의
> fsck는 반드시 **언마운트 상태**에서 실행해야 한다. 마운트된 상태에서 실행 시 데이터 손상 위험.

### e2fsck

**ext 계열** 파일시스템 전용 점검/복구 도구.

```bash
e2fsck [옵션] 장치명
e2fsck /dev/sdb1          # ext 파일시스템 점검
e2fsck -y /dev/sdb1       # 자동 복구
e2fsck -p /dev/sdb1       # 안전하게 자동 복구
e2fsck -f /dev/sdb1       # 강제 점검 (clean해도 점검)
```

| 옵션 | 설명 |
|------|------|
| -y | 모든 질문에 yes |
| -p | 안전한 수준의 자동 복구 |
| -f | 강제 점검 |
| -n | 점검만, 수정 안 함 |

### xfs_repair

**XFS 파일시스템** 전용 복구 도구. XFS에서는 fsck 대신 xfs_repair를 사용한다.

```bash
xfs_repair /dev/sdb1         # XFS 파일시스템 복구
xfs_repair -n /dev/sdb1      # 점검만, 수정 안 함
xfs_repair -L /dev/sdb1      # 로그 초기화 후 복구
```

## du vs df

### du (Disk Usage)

파일/디렉토리의 **디스크 사용량**을 확인한다.

```bash
du [옵션] [파일/디렉토리]
du -sh /home/june      # /home/june의 총 사용량 (사람이 읽기 쉬운 단위)
du -h /var/log         # /var/log 하위 디렉토리별 사용량
du -a /home            # 파일별 사용량 포함
```

| 옵션 | 설명 |
|------|------|
| -s | 요약(총합)만 표시 (summary) |
| -h | 사람이 읽기 쉬운 단위 (KB, MB, GB) |
| -a | 파일별 사용량도 표시 (all) |
| -k | KB 단위 표시 |
| -m | MB 단위 표시 |

### df (Disk Free)

마운트된 파일시스템의 **디스크 여유 공간**을 확인한다.

```bash
df [옵션]
df -h              # 사람이 읽기 쉬운 단위로 표시
df -T              # 파일시스템 유형도 표시
df -hT             # 사람이 읽기 쉬운 단위 + 파일시스템 유형
df -i              # inode 사용량 표시
```

| 옵션 | 설명 |
|------|------|
| -h | 사람이 읽기 쉬운 단위 (human-readable) |
| -T | 파일시스템 유형 표시 (Type) |
| -i | inode 사용 정보 표시 |
| -a | 모든 파일시스템 표시 |
| -k | KB 단위 |

> [!important] du vs df 비교
> | 구분 | du | df |
> |------|-----|-----|
> | 측정 대상 | **사용량** (얼마나 사용했나) | **여유 공간** (얼마나 남았나) |
> | 기본 단위 | 파일/디렉토리 단위 | 파일시스템(파티션) 단위 |
> | 핵심 옵션 | -s(요약), -h | -h, -T(타입) |

## make clean

소스 코드 컴파일 과정에서 **이전 빌드 결과물(오브젝트 파일 등)을 삭제**하는 명령어.

```bash
# 일반적인 소스 컴파일 절차
./configure         # 환경 설정
make                # 컴파일
make install        # 설치
make clean          # 이전 빌드 결과물 정리
```

> [!tip] make clean의 위치
> make clean은 **Makefile**이 있는 디렉토리에서 실행한다.
> 재컴파일 전에 이전 오브젝트 파일(.o)을 제거하여 깨끗한 빌드를 보장한다.
