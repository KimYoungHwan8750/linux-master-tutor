## 디스크 쿼타(Quota)

사용자 또는 그룹별로 디스크 사용량을 **제한**하는 기능.

| 명령어 | 용도 |
|--------|------|
| `quota` | 현재 쿼타 확인 |
| `edquota` | 대화형으로 쿼타 편집 |
| `setquota` | CLI에서 직접 쿼타 설정 |
| `repquota` | 파일시스템 쿼타 보고서 출력 |
| `quotaon` | 쿼타 활성화 |
| `quotaoff` | 쿼타 비활성화 |
| `xfs_quota` | XFS 파일시스템 전용 쿼타 관리 |

> [!tip] 쿼타 설정 흐름
> `/etc/fstab` 수정(usrquota/grpquota 옵션) → 재마운트 → `quotacheck` → `quotaon` → `edquota`

### 쿼타 용어
| 용어 | 의미 |
|------|------|
| soft limit | 경고 발생 한계 (일시적 초과 가능) |
| hard limit | 절대 상한 (초과 불가) |
| grace period | soft limit 초과 후 허용 기간 |

## 파일시스템 종류

| 파일시스템 | 특징 |
|------------|------|
| **ext3** | 저널링 지원, ext2 호환 |
| **ext4** | 대용량 지원, 향상된 저널링, 현재 리눅스 기본 |
| **XFS** | 고성능, 대용량, RHEL 7+ 기본 |
| **NFS** | 네트워크 파일 시스템 (원격 공유) |
| **SMB/CIFS** | Windows 파일 공유 프로토콜 |

> [!warning] ext3 vs ext4 차이
> ext4는 ext3 대비 **최대 파일 크기**(16TB→16EB), **최대 파일시스템 크기**(32TB→1EB)가 크게 향상됨
