## 디스크 쿼터 (Disk Quota) 개념

디스크 쿼터란 **파일시스템별**로 사용자나 그룹이 사용할 수 있는 **디스크 용량 및 파일 수(inode)를 제한**하는 기능이다.

### 쿼터 주요 용어

| 용어 | 설명 |
|------|------|
| 하드 제한 (hard limit) | 절대 초과 불가능한 최대 한도 |
| 소프트 제한 (soft limit) | 일시적으로 초과 허용, 유예 기간(grace period) 내 해결 필요 |
| 유예 기간 (grace period) | 소프트 제한 초과 후 허용되는 기간 (초과 상태 유지 시 쓰기 차단) |
| inode 제한 | 생성 가능한 파일 개수 제한 |
| block 제한 | 사용 가능한 디스크 용량 제한 |

### 쿼터 설정 절차

```
1. /etc/fstab에 usrquota, grpquota 옵션 추가
2. mount -o remount (재마운트)
3. quotacheck -augm (쿼터 파일 생성)
4. edquota 또는 setquota로 쿼터 설정
5. quotaon -a (쿼터 활성화)
```

## 쿼터 명령어 상세

### quota

```bash
quota [옵션] [사용자명]
quota -u june       # june 사용자의 쿼터 확인
quota -g staff      # staff 그룹의 쿼터 확인
```

| 옵션 | 설명 |
|------|------|
| -u | 사용자 쿼터 표시 |
| -g | 그룹 쿼터 표시 |
| -v | 상세 정보 표시 |
| -s | 사람이 읽기 쉬운 단위(KB, MB 등) |

### edquota (편집기 방식)

**편집기(vi)를 사용**하여 쿼터를 설정한다. 인자가 적고 대화식으로 설정한다.

```bash
edquota [옵션] 사용자명
edquota june          # june의 쿼터를 vi 편집기로 설정
edquota -g staff      # staff 그룹 쿼터 설정
edquota -t            # grace period(유예 기간) 설정
edquota -p user1 user2  # user1의 설정을 user2에 복사
```

| 옵션 | 설명 |
|------|------|
| -u | 사용자 쿼터 편집 (기본값) |
| -g | 그룹 쿼터 편집 |
| -t | 유예 기간(grace period) 설정 |
| -p | 다른 사용자의 쿼터를 복사 |

### setquota (명령행 방식)

**명령행에서 직접** 쿼터 값을 지정한다. 스크립트에서 자동화에 유리하다.

```bash
setquota [옵션] 사용자명 block-soft block-hard inode-soft inode-hard 파일시스템
setquota -u june 100000 200000 0 0 /home
```

| 옵션 | 설명 |
|------|------|
| -u | 사용자 쿼터 설정 |
| -g | 그룹 쿼터 설정 |
| -a | 모든 파일시스템에 적용 |

> [!important] edquota vs setquota 비교
> | 구분 | edquota | setquota |
> |------|---------|----------|
> | 설정 방식 | **vi 편집기** 사용 | **명령행**에서 직접 입력 |
> | 인자 | 적음 (편집기에서 입력) | 많음 (block-soft, block-hard, inode-soft, inode-hard) |
> | 자동화 | 어려움 | **스크립트 자동화에 적합** |
> | -u 옵션 | 기본값 (생략 가능) | **명시적으로 사용** |

### xfs_quota

**XFS 파일시스템** 전용 쿼터 관리 도구. CentOS 7 이상에서 기본 파일시스템이 XFS이므로 자주 사용된다.

```bash
xfs_quota -x -c 'limit bsoft=100m bhard=200m june' /home
xfs_quota -x -c 'report -h' /home
```

| 옵션 | 설명 |
|------|------|
| -x | 전문가(expert) 모드 활성화 |
| -c '명령' | 실행할 쿼터 명령을 지정 |

하위 명령어:
| 하위 명령 | 설명 |
|-----------|------|
| limit | 쿼터 제한 설정 |
| report | 쿼터 사용량 보고 |
| quota | 개별 사용자 쿼터 조회 |

### repquota

파일시스템의 **쿼터 사용량을 요약 보고**한다.

```bash
repquota [옵션] 파일시스템
repquota -a       # 모든 파일시스템의 쿼터 보고
repquota -u /home # /home의 사용자 쿼터 보고
repquota -g /home # /home의 그룹 쿼터 보고
repquota -s /home # 사람이 읽기 쉬운 단위로 표시
```

### quotaon / quotaoff

```bash
quotaon [옵션] 파일시스템
quotaon -a        # 모든 파일시스템의 쿼터 활성화
quotaon -u /home  # /home의 사용자 쿼터 활성화
quotaoff -a       # 모든 파일시스템의 쿼터 비활성화
```

### quotacheck

쿼터 파일을 **생성하고 점검**한다. (aquota.user, aquota.group 파일 생성)

```bash
quotacheck [옵션] 파일시스템
quotacheck -augm    # 가장 많이 쓰는 조합
```

| 옵션 | 설명 |
|------|------|
| -a | 모든 파일시스템 점검 |
| -u | 사용자 쿼터 점검 |
| -g | 그룹 쿼터 점검 |
| -m | 마운트된 상태에서 강제 점검 (remount 하지 않음) |

## 리눅스 파일시스템 종류

### ext 계열 파일시스템

| 파일시스템 | 특징 |
|-----------|------|
| **ext** | 리눅스 초기 파일시스템, 최대 2GB |
| **ext2** | ext 개선, 최대 파일 크기 2GB, 파일시스템 크기 4TB, **저널링 미지원** |
| **ext3** | ext2 + **저널링(Journaling)** 지원, **ACL(Access Control List)** 지원, ext2와 호환 |
| **ext4** | ext3 개선, 최대 파일 크기 16TB, 파일시스템 크기 1EB, 하위 호환성 유지, 지연 할당 |

### 저널링 (Journaling) 개념

> [!important] 저널링이란?
> 파일시스템 변경 사항을 **저널(로그)**에 먼저 기록한 후 실제 디스크에 반영하는 기술.
> **비정상 종료(정전, 시스템 크래시) 시** 저널 로그를 이용하여 **빠르고 안정적인 복구**가 가능하다.
> ext2는 저널링 미지원 → 비정상 종료 시 전체 파일시스템 검사 필요 (느림)
> ext3부터 저널링 지원 → 저널 로그만 확인하면 되므로 복구가 빠름

### 기타 리눅스 파일시스템

| 파일시스템 | 특징 |
|-----------|------|
| **XFS** | SGI 개발, 대용량 파일 처리에 우수, **CentOS 7 기본 파일시스템**, 저널링 지원 |
| **Btrfs** | Oracle 개발, 스냅샷/압축/RAID 지원, 차세대 파일시스템 |
| **JFS** | IBM 개발, 저널링 지원, AIX에서 이식 |
| **ReiserFS** | 저널링 지원, 소형 파일 처리에 효율적 |
| **ISO 9660** | CD-ROM 표준 파일시스템 |
| **UDF** | DVD 표준 파일시스템 |

## 네트워크 파일시스템

| 파일시스템 | 설명 |
|-----------|------|
| **SMB** (Server Message Block) | **삼바(Samba)** 프로토콜, **윈도우-리눅스 간 파일/프린터 공유**, Microsoft 개발 |
| **CIFS** (Common Internet File System) | SMB를 확장/개선한 프로토콜, 인터넷을 통한 파일 공유 |
| **NFS** (Network File System) | **유닉스/리눅스 간 네트워크 파일 공유**, Sun Microsystems 개발, RPC 기반 |

> [!tip] SMB vs NFS 구분
> - **SMB/CIFS** → **윈도우**와의 파일 공유 (삼바)
> - **NFS** → **리눅스/유닉스** 간 파일 공유

## /etc/fstab 구조

시스템 부팅 시 **자동 마운트**할 파일시스템 정보를 담고 있는 설정 파일.

```
# 장치명      마운트포인트  파일시스템  옵션      덤프  fsck순서
/dev/sda1    /            ext4       defaults  1     1
/dev/sda2    /home        ext4       defaults  1     2
UUID=xxxx    /boot        xfs        defaults  0     0
/dev/sda3    swap         swap       defaults  0     0
```

| 필드 | 설명 | 예시 |
|------|------|------|
| 1. 장치명 | 장치 파일 경로 또는 UUID | /dev/sda1, UUID=xxxx |
| 2. 마운트 포인트 | 마운트할 디렉토리 | /, /home, /boot |
| 3. 파일시스템 유형 | 파일시스템 종류 | ext4, xfs, swap, nfs |
| 4. 옵션 | 마운트 옵션 | defaults, ro, noexec, usrquota |
| 5. 덤프 | dump 백업 여부 (0=안 함, 1=함) | 0, 1 |
| 6. fsck 순서 | 부팅 시 파일시스템 점검 순서 (0=안 함, 1=루트, 2=기타) | 0, 1, 2 |

### fstab 주요 마운트 옵션

| 옵션 | 설명 |
|------|------|
| defaults | rw, suid, dev, exec, auto, nouser, async 포함 |
| ro | 읽기 전용 |
| rw | 읽기/쓰기 |
| noexec | 실행 파일 실행 불가 |
| nosuid | SetUID/SetGID 무시 |
| auto | 부팅 시 자동 마운트 |
| noauto | 부팅 시 자동 마운트 안 함 |
| usrquota | 사용자 쿼터 활성화 |
| grpquota | 그룹 쿼터 활성화 |

## UUID와 blkid

### UUID (Universally Unique Identifier)

파티션마다 부여되는 **고유한 128비트 식별자**. 장치 파일명(/dev/sdX)은 장치 순서에 따라 바뀔 수 있지만, UUID는 **변하지 않으므로** /etc/fstab에서 안정적인 식별에 사용된다.

### blkid 명령어

블록 장치의 UUID, 파일시스템 유형 등을 확인한다.

```bash
blkid                    # 모든 블록 장치의 UUID 및 정보 표시
blkid /dev/sda1          # 특정 장치의 UUID 확인
```

출력 예:
```
/dev/sda1: UUID="a1b2c3d4-..." TYPE="ext4" PARTUUID="xxxx"
```
