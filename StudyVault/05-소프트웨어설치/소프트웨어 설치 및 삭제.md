## 한눈에 비교

### 패키지 관리 체계 총정리

| 구분 | Red Hat 계열 | Debian 계열 | SUSE |
|------|-------------|-------------|------|
| **오프라인(로컬)** | `rpm` | `dpkg` | `rpm` |
| **온라인(저장소)** | `yum` → `dnf` | `apt-get` / `apt` | `zypper` |
| **패키지 확장자** | `.rpm` | `.deb` | `.rpm` |
| **의존성 해결** | 온라인만 자동 | 온라인만 자동 | 온라인만 자동 |

### 압축 효율성 순서

```
xz > bz2 > gz > compress
(가장 높음)              (가장 낮음)
확장자: .xz   .bz2   .gz   .Z
```

## 1. 온라인 패키지 관리자

> [!info] 온라인 패키지 관리자는 **인터넷 저장소(Repository)**에서 패키지를 다운로드하고 **의존성을 자동으로 해결**하여 설치한다.

### 1-1. yum (Yellow dog Updater, Modified) — Red Hat 계열

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `yum install 패키지` | 패키지 설치 | `yum install httpd` |
| `yum remove 패키지` | 패키지 삭제 | `yum remove httpd` |
| `yum update` | 전체 업데이트 | `yum update` |
| `yum update 패키지` | 특정 패키지 업데이트 | `yum update httpd` |
| `yum search 키워드` | 패키지 검색 | `yum search web` |
| `yum info 패키지` | 패키지 정보 확인 | `yum info httpd` |
| `yum list` | 전체 패키지 목록 | `yum list installed` |
| `yum groupinstall 그룹` | 그룹 패키지 설치 | `yum groupinstall "Development Tools"` |
| `yum check-update` | 업데이트 가능 목록 | `yum check-update` |
| `yum clean all` | 캐시 삭제 | `yum clean all` |

- **설정 파일**: `/etc/yum.conf`, `/etc/yum.repos.d/*.repo`
- `-y` 옵션: 모든 질문에 자동으로 yes 응답

### 1-2. apt-get — Debian 계열 (Ubuntu, Debian)

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `apt-get install 패키지` | 패키지 설치 | `apt-get install apache2` |
| `apt-get remove 패키지` | 패키지 삭제 (설정 유지) | `apt-get remove apache2` |
| `apt-get purge 패키지` | 패키지 + 설정 완전 삭제 | `apt-get purge apache2` |
| `apt-get update` | 패키지 목록 갱신 | `apt-get update` |
| `apt-get upgrade` | 설치된 패키지 업그레이드 | `apt-get upgrade` |
| `apt-get dist-upgrade` | 의존성 고려한 전체 업그레이드 | `apt-get dist-upgrade` |
| `apt-cache search 키워드` | 패키지 검색 | `apt-cache search web` |
| `apt-cache show 패키지` | 패키지 정보 | `apt-cache show apache2` |

> [!warning] apt-get의 remove vs purge
> - `remove`: 패키지만 삭제, **설정 파일은 남김**
> - `purge`: 패키지 + **설정 파일까지 완전 삭제**

- **설정 파일**: `/etc/apt/sources.list`
- `apt-get update`는 설치가 아니라 **패키지 목록을 최신으로 갱신**하는 명령
- `upgrade` vs `dist-upgrade`: dist-upgrade는 의존성 변경 시 **패키지 추가/삭제**도 수행

### 1-3. dnf (Dandified YUM) — Fedora, 최신 Red Hat 계열

- **yum의 후속** 패키지 관리자 (Fedora 22+, RHEL 8+)
- yum과 **동일한 명령 체계** 사용 가능
- 개선점: 속도 향상, 의존성 해결 알고리즘 개선, Python 3 기반

| 명령어 | 설명 |
|--------|------|
| `dnf install 패키지` | 패키지 설치 |
| `dnf remove 패키지` | 패키지 삭제 |
| `dnf update` | 전체 업데이트 |
| `dnf search 키워드` | 패키지 검색 |
| `dnf info 패키지` | 패키지 정보 |
| `dnf list` | 패키지 목록 |
| `dnf groupinstall 그룹` | 그룹 설치 |

### 1-4. zypper — SUSE 계열

| 명령어 | 설명 |
|--------|------|
| `zypper install 패키지` | 패키지 설치 |
| `zypper remove 패키지` | 패키지 삭제 |
| `zypper update` | 업데이트 |
| `zypper search 키워드` | 검색 |
| `zypper info 패키지` | 정보 확인 |
| `zypper refresh` | 저장소 메타데이터 갱신 |

### 온라인 패키지 관리자 비교표

| 기능 | yum/dnf (Red Hat) | apt-get (Debian) | zypper (SUSE) |
|------|-------------------|------------------|---------------|
| 설치 | `install` | `install` | `install` |
| 삭제 | `remove` | `remove` / `purge` | `remove` |
| 업데이트 | `update` | `upgrade` | `update` |
| 목록 갱신 | 자동 | `update` | `refresh` |
| 검색 | `search` | `apt-cache search` | `search` |
| 정보 | `info` | `apt-cache show` | `info` |
| 그룹 설치 | `groupinstall` | tasksel | pattern |

## 2. 오프라인 패키지 관리자

> [!info] 오프라인 패키지 관리자는 **로컬에 있는 패키지 파일**을 직접 설치하며, 의존성을 **자동으로 해결하지 않는다**.

### 2-1. rpm (Red Hat Package Manager)

**패키지 이름 형식**: `패키지이름-버전-릴리즈.아키텍처.rpm`
- 예: `httpd-2.4.6-90.el7.x86_64.rpm`

#### rpm 주요 옵션

| 모드 | 옵션 | 설명 | 예시 |
|------|------|------|------|
| **설치** | `-i` | 새로 설치 | `rpm -ivh httpd.rpm` |
| **설치** | `-ivh` | 설치 + 상세 + 진행 | `rpm -ivh httpd.rpm` |
| **업그레이드** | `-U` | 업그레이드 (없으면 설치) | `rpm -Uvh httpd.rpm` |
| **업그레이드** | `-F` | 존재할 때만 업그레이드 | `rpm -Fvh httpd.rpm` |
| **삭제** | `-e` | 삭제 | `rpm -e httpd` |
| **질의** | `-q` | 패키지 설치 여부 확인 | `rpm -q httpd` |
| **질의** | `-qa` | 설치된 전체 패키지 목록 | `rpm -qa` |
| **질의** | `-qi` | 패키지 상세 정보 | `rpm -qi httpd` |
| **질의** | `-ql` | 패키지 파일 목록 | `rpm -ql httpd` |
| **질의** | `-qf 파일` | 파일이 속한 패키지 확인 | `rpm -qf /etc/httpd/conf/httpd.conf` |
| **질의** | `-qR` | 의존 패키지 목록 | `rpm -qR httpd` |
| **검증** | `-V` | 패키지 파일 변경 검증 | `rpm -V httpd` |
| **검증** | `-Va` | 모든 패키지 검증 | `rpm -Va` |

> [!tip] rpm 설치 옵션 기억법
> - `-i` (install): 새로 설치
> - `-U` (Upgrade): 업그레이드, **없으면 설치**도 함
> - `-F` (Freshen): **이미 설치된 것만** 업그레이드
> - 보통 `-vh` 옵션을 함께 사용 (v: verbose, h: hash 진행표시)

#### rpm 검증(-V) 결과 코드

| 코드 | 의미 |
|------|------|
| `S` | 파일 크기(Size) 변경 |
| `M` | 모드(Mode, 권한) 변경 |
| `5` | MD5 체크섬 변경 |
| `D` | 장치(Device) 변경 |
| `L` | 심볼릭 링크 변경 |
| `U` | 소유자(User) 변경 |
| `G` | 그룹(Group) 변경 |
| `T` | 수정 시간(Time) 변경 |

### 2-2. dpkg (Debian Package)

**패키지 이름 형식**: `패키지이름_버전_아키텍처.deb`
- 예: `apache2_2.4.41-4ubuntu3_amd64.deb`

#### dpkg 주요 옵션

| 옵션 | 설명 | 예시 |
|------|------|------|
| `-i` | 패키지 설치 (install) | `dpkg -i apache2.deb` |
| `-r` | 패키지 삭제 (remove, 설정 유지) | `dpkg -r apache2` |
| `-P` | 완전 삭제 (Purge, 설정 포함) | `dpkg -P apache2` |
| `-l` | 설치된 패키지 목록 (list) | `dpkg -l` |
| `-s` | 패키지 상태 확인 (status) | `dpkg -s apache2` |
| `-L` | 패키지 설치 파일 목록 (List files) | `dpkg -L apache2` |
| `-S 파일` | 파일이 속한 패키지 확인 | `dpkg -S /etc/apache2/apache2.conf` |
| `-c` | .deb 파일 내용 확인 | `dpkg -c apache2.deb` |
| `--configure` | 설정되지 않은 패키지 구성 | `dpkg --configure -a` |

### rpm vs dpkg 비교표

| 기능 | rpm (Red Hat) | dpkg (Debian) |
|------|--------------|---------------|
| 설치 | `-i` (install) | `-i` (install) |
| 업그레이드 | `-U` (Upgrade) | `-i` (재설치) |
| 존재시만 업그레이드 | `-F` (Freshen) | — |
| 삭제 | `-e` (erase) | `-r` (remove) |
| 완전 삭제 (설정 포함) | `-e` | `-P` (Purge) |
| 전체 목록 | `-qa` | `-l` |
| 패키지 정보 | `-qi` | `-s` |
| 파일 목록 | `-ql` | `-L` |
| 파일→패키지 | `-qf` | `-S` |
| 검증 | `-V` | — |

## 3. 소스 코드 설치 순서

> [!important] 시험 빈출! 소스 코드 컴파일 설치 순서를 반드시 기억하자.

```
┌─────────────────────────────────────────────────────────┐
│  소스 코드 설치 5단계                                    │
│                                                         │
│  1. tar xvf 소스파일.tar.gz     ← 압축 해제             │
│          ↓                                              │
│  2. ./configure                 ← 환경설정, Makefile 생성│
│          ↓                                              │
│  3. make                        ← 소스 코드 컴파일       │
│          ↓                                              │
│  4. make install                ← 바이너리 설치          │
│          ↓                                              │
│  5. make clean                  ← 이전 설정/파일 삭제    │
└─────────────────────────────────────────────────────────┘
```

### 각 단계 상세

| 단계 | 명령어 | 역할 | 비고 |
|------|--------|------|------|
| 1 | `tar xvzf source.tar.gz` | 압축 해제 | 디렉터리 이동 후 진행 |
| 2 | `./configure` | 시스템 환경 검사, **Makefile** 생성 | `--prefix=/usr/local` 등 옵션 |
| 3 | `make` | Makefile을 읽어 소스 코드 **컴파일** | gcc 등 컴파일러 사용 |
| 4 | `make install` | 컴파일된 바이너리를 시스템에 **설치** | 보통 root 권한 필요 |
| 5 | `make clean` | 컴파일 과정에서 생성된 **오브젝트 파일 삭제** | 재컴파일 시 사용 |

> [!warning] configure 실패 시
> - 필요한 라이브러리나 개발 도구가 없으면 configure 단계에서 오류 발생
> - 오류 메시지를 확인하고 필요한 패키지를 먼저 설치해야 함
> - `cmake`는 ./configure 대신 사용하는 빌드 시스템 (`cmake .` → `make` → `make install`)

## 4. 압축 명령어

### 4-1. tar (Tape ARchive)

> [!info] tar 자체는 **아카이브(묶기)** 도구이며, 압축은 별도 옵션으로 지정한다.

#### tar 주요 옵션

| 옵션 | 의미 | 설명 |
|------|------|------|
| `-c` | create | 아카이브 **생성** |
| `-x` | extract | 아카이브 **해제** |
| `-t` | list | 아카이브 **내용 확인** |
| `-v` | verbose | **상세 정보** 출력 |
| `-f` | file | 아카이브 **파일명** 지정 (필수) |
| `-z` | gzip | **gzip** 압축/해제 (.tar.gz, .tgz) |
| `-j` | bzip2 | **bzip2** 압축/해제 (.tar.bz2) |
| `-J` | xz | **xz** 압축/해제 (.tar.xz) |
| `-Z` | compress | **compress** 압축/해제 (.tar.Z) |
| `-p` | permission | 권한 유지 |
| `-C` | directory | 특정 디렉터리에 풀기 |

> [!tip] 압축 옵션 기억법
> ```
> -z (gzip)     : z는 gz의 마지막 글자
> -j (bzip2)    : j는 bz2와 비슷한 발음
> -J (xz)       : 대문자 J는 xz (더 강력)
> -Z (compress) : 대문자 Z는 .Z 파일
> ```

#### tar 사용 예시

```bash
# 생성 (압축)
tar cvf archive.tar dir/        # 묶기만 (압축 없음)
tar cvzf archive.tar.gz dir/    # gzip 압축
tar cvjf archive.tar.bz2 dir/   # bzip2 압축
tar cvJf archive.tar.xz dir/    # xz 압축

# 해제
tar xvf archive.tar             # 묶기 해제
tar xvzf archive.tar.gz         # gzip 해제
tar xvjf archive.tar.bz2        # bzip2 해제
tar xvJf archive.tar.xz         # xz 해제

# 내용 확인
tar tvf archive.tar             # 아카이브 내용 보기

# 특정 디렉터리에 풀기
tar xvzf archive.tar.gz -C /opt/
```

### 4-2. 개별 압축 명령어

#### gzip / gunzip (.gz)

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `gzip 파일` | 파일 압축 (.gz 생성, 원본 삭제) | `gzip file.txt` |
| `gzip -d 파일.gz` | 압축 해제 | `gzip -d file.txt.gz` |
| `gunzip 파일.gz` | 압축 해제 (= gzip -d) | `gunzip file.txt.gz` |
| `gzip -r 디렉터리` | 디렉터리 내 파일 재귀 압축 | `gzip -r /data/` |
| `gzip -l 파일.gz` | 압축 정보 확인 | `gzip -l file.txt.gz` |
| `gzip -1 ~ -9` | 압축 레벨 (1=빠름, 9=높은압축) | `gzip -9 file.txt` |
| `zcat 파일.gz` | 압축 해제 없이 내용 보기 | `zcat file.txt.gz` |

#### bzip2 / bunzip2 (.bz2)

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `bzip2 파일` | 파일 압축 (.bz2 생성) | `bzip2 file.txt` |
| `bzip2 -d 파일.bz2` | 압축 해제 | `bzip2 -d file.txt.bz2` |
| `bunzip2 파일.bz2` | 압축 해제 (= bzip2 -d) | `bunzip2 file.txt.bz2` |
| `bzip2 -k 파일` | 원본 유지하며 압축 | `bzip2 -k file.txt` |
| `bzcat 파일.bz2` | 압축 해제 없이 내용 보기 | `bzcat file.txt.bz2` |

#### xz / unxz (.xz)

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `xz 파일` | 파일 압축 (.xz 생성) | `xz file.txt` |
| `xz -d 파일.xz` | 압축 해제 | `xz -d file.txt.xz` |
| `unxz 파일.xz` | 압축 해제 (= xz -d) | `unxz file.txt.xz` |
| `xz -k 파일` | 원본 유지하며 압축 | `xz -k file.txt` |
| `xzcat 파일.xz` | 압축 해제 없이 내용 보기 | `xzcat file.txt.xz` |

#### compress / uncompress (.Z)

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `compress 파일` | 파일 압축 (.Z 생성) | `compress file.txt` |
| `uncompress 파일.Z` | 압축 해제 | `uncompress file.txt.Z` |
| `zcat 파일.Z` | 압축 해제 없이 내용 보기 | `zcat file.txt.Z` |

### 압축 명령어 종합 비교표

| 압축도구 | 확장자 | 해제 명령 | 내용 보기 | tar 옵션 | 압축 효율 |
|----------|--------|-----------|-----------|----------|-----------|
| `compress` | `.Z` | `uncompress` | `zcat` | `-Z` | ★ (낮음) |
| `gzip` | `.gz` | `gunzip` / `gzip -d` | `zcat` | `-z` | ★★ |
| `bzip2` | `.bz2` | `bunzip2` / `bzip2 -d` | `bzcat` | `-j` | ★★★ |
| `xz` | `.xz` | `unxz` / `xz -d` | `xzcat` | `-J` | ★★★★ (높음) |

## 시험 빈출 패턴

| 시나리오/키워드 | 정답 |
|----------------|------|
| rpm으로 패키지 설치 (상세+진행) | `rpm -ivh 패키지.rpm` |
| rpm 업그레이드 (없으면 설치) | `rpm -Uvh 패키지.rpm` |
| rpm 이미 설치된 것만 업그레이드 | `rpm -Fvh 패키지.rpm` |
| rpm 삭제 | `rpm -e 패키지` |
| rpm 설치된 전체 패키지 | `rpm -qa` |
| rpm 파일이 속한 패키지 | `rpm -qf /경로/파일` |
| dpkg 설정 포함 완전 삭제 | `dpkg -P 패키지` |
| dpkg 설치 파일 목록 | `dpkg -L 패키지` |
| apt-get 패키지 목록 갱신 | `apt-get update` |
| apt-get remove vs purge | remove: 설정 유지 / purge: 설정도 삭제 |
| 소스 설치 순서 | tar → configure → make → make install |
| configure 역할 | 환경설정, **Makefile** 생성 |
| 압축 효율 순서 | xz > bz2 > gz > compress |
| tar에서 bzip2 옵션 | `-j` |
| tar에서 xz 옵션 | `-J` (대문자) |
| yum의 후속 패키지 관리자 | `dnf` |
