## 프로세스 개념

### 프로세스란?

실행 중인 프로그램의 인스턴스. 각 프로세스는 고유한 **PID(Process ID)**를 가진다.

| 용어 | 설명 |
|------|------|
| PID | 프로세스 고유 식별 번호 |
| PPID | 부모 프로세스 ID |
| UID | 프로세스를 실행한 사용자 ID |
| init/systemd | PID 1번, 모든 프로세스의 최상위 부모 |

### 포그라운드 vs 백그라운드

| 구분 | 포그라운드 (Foreground) | 백그라운드 (Background) |
|------|----------------------|----------------------|
| 실행 방식 | 터미널에서 직접 실행 | 명령어 뒤에 **&** 붙여 실행 |
| 사용자 상호작용 | **대화형** (입력 대기) | 터미널 제어 반환 (다른 작업 가능) |
| 예시 | `./script.sh` | `./script.sh &` |

```bash
./long_task.sh &        # 백그라운드로 실행
jobs                     # 백그라운드 작업 목록
fg %1                    # 작업 1을 포그라운드로 전환
# Ctrl+Z                 # 포그라운드 작업 일시 정지
bg %1                    # 정지된 작업을 백그라운드에서 재개
```

## 데몬 (Daemon)

백그라운드에서 **서비스를 제공**하는 프로세스. 사용자와 직접 상호작용하지 않는다.

### standalone vs inetd/xinetd

| 구분 | standalone 방식 | inetd/xinetd 방식 |
|------|----------------|-------------------|
| 동작 방식 | **메모리에 상주**, 항상 실행 | **요청 시 생성**, 평소 미실행 |
| 응답 속도 | **즉시 응답** (빠름) | 프로세스 생성 후 응답 (느림) |
| 메모리 사용 | **많음** (항상 점유) | **효율적** (필요시만 점유) |
| 적용 대상 | 자주 사용하는 서비스 (httpd, sshd, mysqld) | 가끔 사용하는 서비스 (telnet, finger) |
| 관리 | 개별 데몬이 스스로 관리 | inetd/xinetd가 **중앙에서 관리** |

```
standalone 방식:
┌─────────┐  요청  ┌─────────┐
│ 클라이언트 │──────→│ httpd   │  ← 항상 메모리에 상주
└─────────┘       └─────────┘

inetd/xinetd 방식:
┌─────────┐  요청  ┌──────────┐  생성  ┌─────────┐
│ 클라이언트 │──────→│ xinetd   │──────→│ telnetd │  ← 요청 시 생성
└─────────┘       └──────────┘       └─────────┘
                   (슈퍼 데몬)
```

## 프로세스 관리 명령어

### ps 명령어

현재 실행 중인 프로세스 목록을 표시한다.

```bash
ps                   # 현재 터미널의 프로세스
ps -e                # 시스템의 모든 프로세스 (every)
ps -f                # 풀 포맷 출력 (full)
ps -ef               # 모든 프로세스, 풀 포맷 ★
ps aux               # BSD 스타일 모든 프로세스 ★
ps -u june           # june 사용자의 프로세스
ps -p 1234           # PID 1234 프로세스 정보
```

| 옵션 | 설명 |
|------|------|
| -e | 모든(every) 프로세스 표시 |
| -f | 풀(full) 포맷 출력 (UID, PID, PPID, C, STIME, TTY, TIME, CMD) |
| -a | 터미널과 관련된 모든 프로세스 |
| -u | 사용자 기반 출력 (BSD: 특정 사용자) |
| -x | 터미널 없는 프로세스도 표시 (BSD) |
| -l | 긴 포맷 (long) 출력 |
| -p PID | 특정 PID 프로세스 정보 |

> [!important] ps -ef vs ps aux
> | ps -ef (System V 스타일) | ps aux (BSD 스타일) |
> |-------------------------|---------------------|
> | `-e`: 모든 프로세스 | `a`: 모든 사용자 프로세스 |
> | `-f`: 풀 포맷 | `u`: 사용자 중심 포맷 |
> | | `x`: 터미널 없는 프로세스 포함 |
> | UID, PID, PPID 등 표시 | USER, PID, %CPU, %MEM 등 표시 |

### pstree 명령어

프로세스 간의 **부모-자식 관계를 트리 구조**로 표시한다.

```bash
pstree              # 프로세스 트리 표시
pstree -p           # PID도 함께 표시
pstree -u           # 사용자명도 표시
pstree june         # june 사용자의 프로세스 트리
```

### top 명령어

**실시간**으로 프로세스 상태를 모니터링한다.

```bash
top                  # 실시간 모니터링 시작
top -d 5             # 5초 간격으로 갱신
top -u june          # june 사용자의 프로세스만
top -p 1234          # 특정 PID만 모니터링
```

**top 실행 중 내부 키**:
| 키 | 설명 |
|----|------|
| q | 종료 |
| k | 프로세스 종료 (kill) |
| r | 우선순위 변경 (renice) |
| P | CPU 사용량 기준 정렬 |
| M | 메모리 사용량 기준 정렬 |
| 1 | 각 CPU 코어별 사용량 표시 |

## 작업 제어 (Job Control)

### jobs 명령어

현재 셸의 **백그라운드 작업 목록**을 표시한다.

```bash
jobs                 # 작업 목록 표시
jobs -l              # PID도 함께 표시
```

출력 예:
```
[1]+  Running    ./script1.sh &
[2]-  Stopped    ./script2.sh
```

| 기호 | 의미 |
|------|------|
| + | 현재(최근) 작업 |
| - | 이전 작업 |

### bg / fg 명령어

```bash
bg %1                # 작업 1을 백그라운드에서 재개
fg %1                # 작업 1을 포그라운드로 전환
fg                   # 가장 최근 작업을 포그라운드로
```

## kill / killall 명령어

### kill (PID로 시그널 전송)

```bash
kill [시그널] PID
kill 1234              # PID 1234에 SIGTERM(15) 전송 (기본값)
kill -9 1234           # PID 1234를 강제 종료 (SIGKILL)
kill -15 1234          # 정상 종료 요청 (SIGTERM)
kill -HUP 1234         # 재시작 (SIGHUP)
kill -l                # 시그널 목록 표시
```

### killall (이름으로 시그널 전송)

```bash
killall [시그널] 프로세스명
killall httpd          # httpd 프로세스 모두 종료
killall -9 firefox     # firefox 강제 종료
killall -u june        # june 사용자의 모든 프로세스 종료
```

## 시그널 번호표 ★★★

| 번호 | 시그널명 | 의미 | 키보드 | 비고 |
|------|---------|------|--------|------|
| **1** | **SIGHUP** | 재시작 (Hangup) | | 데몬 설정 다시 읽기 |
| **2** | **SIGINT** | 인터럽트 (종료) | **Ctrl+C** | 포그라운드 프로세스 종료 |
| **9** | **SIGKILL** | **강제 종료** | | **무시/차단 불가능** |
| **15** | **SIGTERM** | **정상 종료** | | **kill 기본값**, 프로세스가 정리 후 종료 |
| **18** | **SIGCONT** | 계속 (Continue) | | 정지된 프로세스 재개 |
| **19** | **SIGSTOP** | 정지 (Stop) | | **무시/차단 불가능** |
| **20** | **SIGTSTP** | 일시 정지 | **Ctrl+Z** | 포그라운드 프로세스 일시 정지 |

> [!warning] 반드시 암기
> - **SIGKILL(9)**: 강제 종료, 무시/차단 **불가능**
> - **SIGTERM(15)**: 정상 종료, **kill 명령의 기본 시그널**
> - **SIGSTOP(19)**: 정지, 무시/차단 **불가능**
> - **Ctrl+C** = SIGINT(2), **Ctrl+Z** = SIGTSTP(20)

## nice / renice 명령어

프로세스의 **우선순위(nice 값)**를 설정/변경한다.

### nice 값 범위

```
 높은 우선순위                              낮은 우선순위
 ←──────────────────────────────────────────→
 -20  -10   0   10   19
 (가장 높음)  (기본값)  (가장 낮음)
```

| 항목 | 설명 |
|------|------|
| 범위 | **-20 ~ 19** |
| 기본값 | **0** |
| 숫자가 낮을수록 | **우선순위 높음** |
| 숫자가 높을수록 | **우선순위 낮음** |
| root만 가능 | **음수값** 설정 (우선순위 높이기) |
| 일반 사용자 | **양수값**만 설정 가능 (우선순위 낮추기만) |

### nice 명령어

프로세스를 **지정된 nice 값으로 시작**한다.

```bash
nice [옵션] 명령어
nice -n 10 ./script.sh     # nice 값 10으로 실행
nice -n -5 ./script.sh     # nice 값 -5로 실행 (root만)
nice ./script.sh            # nice 값 10으로 실행 (기본 증가값)
```

### renice 명령어

**실행 중인** 프로세스의 nice 값을 변경한다.

```bash
renice [옵션] PID
renice -n 5 1234           # PID 1234의 nice 값을 5로 변경
renice -n -10 1234         # PID 1234의 nice 값을 -10으로 (root만)
renice -n 5 -u june        # june 사용자의 모든 프로세스 변경
```

> [!important] nice vs renice
> | 구분 | nice | renice |
> |------|------|--------|
> | 시점 | 프로세스 **시작 시** 우선순위 지정 | **실행 중** 프로세스의 우선순위 변경 |
> | 대상 | 명령어 | PID |

## nohup 명령어

사용자가 **로그아웃 후에도** 프로세스가 계속 실행되도록 한다.
SIGHUP(1번 시그널)을 무시하도록 설정한다.

```bash
nohup 명령어 &
nohup ./long_task.sh &      # 로그아웃 후에도 실행 계속
nohup ./script.sh > output.log 2>&1 &  # 출력을 파일로 저장
```

> [!tip] nohup 특징
> - 출력이 기본적으로 **nohup.out** 파일에 저장됨
> - 보통 **&**(백그라운드)와 함께 사용
> - SIGHUP 시그널을 무시하므로 터미널 종료/로그아웃에도 프로세스 유지

## cron과 crontab

### cron 데몬

**반복적인 예약 작업**을 수행하는 데몬. crond 서비스가 백그라운드에서 실행된다.

### crontab 명령어

사용자별 cron 작업을 관리한다.

```bash
crontab -e               # cron 작업 편집 (vi 편집기)
crontab -l               # 현재 사용자의 cron 목록 확인
crontab -r               # 현재 사용자의 cron 작업 모두 삭제
crontab -u june -e       # june 사용자의 cron 편집 (root만)
```

| 옵션 | 설명 |
|------|------|
| -e | 편집 (edit) |
| -l | 목록 (list) |
| -r | 삭제 (remove) |
| -u 사용자 | 특정 사용자 지정 (root만 가능) |

### crontab 형식

```
분  시  일  월  요일  명령어
*   *   *   *   *     명령어
```

| 필드 | 범위 | 설명 |
|------|------|------|
| 분 | 0-59 | 분 |
| 시 | 0-23 | 시간 (24시간) |
| 일 | 1-31 | 날짜 |
| 월 | 1-12 | 월 |
| 요일 | 0-7 | 요일 (0과 7=일요일, 1=월요일) |

**특수 문자**:
| 문자 | 의미 | 예시 |
|------|------|------|
| * | 모든 값 | * = 매분/매시/매일 |
| , | 여러 값 나열 | 1,3,5 = 1, 3, 5 |
| - | 범위 | 1-5 = 1부터 5까지 |
| / | 간격 | */5 = 5 간격 |

### crontab 예시

```bash
# 매일 오후 1시에 실행
0 13 * * * /home/june/backup.sh

# 매달 5일 자정에 실행
0 0 5 * * /home/june/monthly.sh

# 매주 월요일 자정에 실행
0 0 * * 1 /home/june/weekly.sh

# 5분마다 실행
*/5 * * * * /home/june/check.sh

# 월~금 오전 9시에 실행
0 9 * * 1-5 /home/june/work.sh

# 1월과 7월 1일 자정에 실행
0 0 1 1,7 * /home/june/semiannual.sh

# 매시 30분에 실행
30 * * * * /home/june/hourly.sh
```

### cron 관련 파일

| 파일/디렉토리 | 설명 |
|-------------|------|
| /etc/crontab | 시스템 전체 cron 설정 |
| /var/spool/cron/ | 사용자별 crontab 파일 저장 위치 |
| /etc/cron.allow | cron 사용 허용 목록 |
| /etc/cron.deny | cron 사용 거부 목록 |

> [!important] allow / deny 규칙
> - /etc/cron.allow가 있으면: **목록에 있는 사용자만** 사용 가능
> - /etc/cron.allow가 없고 /etc/cron.deny가 있으면: **목록에 없는 사용자만** 사용 가능
> - 둘 다 없으면: **root만** 사용 가능 (배포판에 따라 다름)

## at 명령어

**일회성 예약 작업**을 설정한다 (cron은 반복, at은 1회).

```bash
at [시간]
at 3pm                    # 오후 3시에 실행
at 15:00                  # 오후 3시에 실행
at now + 1 hour           # 1시간 후 실행
at now + 30 minutes       # 30분 후 실행
at 10:00 tomorrow         # 내일 오전 10시
at midnight               # 자정
```

```bash
# at 작업 입력 예시
at 15:00
at> /home/june/backup.sh
at> ctrl+D                # 입력 종료
```

| 명령어 | 설명 |
|--------|------|
| at | 일회성 예약 작업 설정 |
| atq | 예약된 at 작업 목록 확인 |
| atrm 작업번호 | 예약된 at 작업 삭제 |

| 관련 파일 | 설명 |
|-----------|------|
| /etc/at.allow | at 사용 허용 목록 |
| /etc/at.deny | at 사용 거부 목록 |
