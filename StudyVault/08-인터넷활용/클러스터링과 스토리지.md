# 클러스터링과 스토리지 (★★)

#network #storage #linux-master

## Overview Table (한눈에 비교)
| 기술 | 목적 |
|------|------|
| 클러스터링 | 여러 서버를 하나처럼 묶어 성능/가용성 향상 |
| RAID | 여러 디스크를 묶어 성능/안정성 향상 |
| LVM | 논리적 볼륨 관리로 유연한 디스크 사용 |

## 클러스터링

| 유형 | 목적 | 설명 |
|------|------|------|
| **HPC** (High Performance Computing) | 성능 | 과학 계산, 대규모 연산 (Beowulf) |
| **LVS** (Linux Virtual Server) | 부하 분산 | 네트워크 트래픽 분산 |
| **HA** (High Availability) | 고가용성 | 장애 발생 시 자동 전환 |

## 클라우드 컴퓨팅

| 유형 | 제공 범위 | 예시 |
|------|-----------|------|
| **IaaS** | 인프라 (서버, 스토리지, 네트워크) | AWS EC2, OpenStack |
| **PaaS** | 플랫폼 (런타임, 미들웨어) | Heroku, Google App Engine |
| **SaaS** | 소프트웨어 (완성된 앱) | Gmail, Google Docs |

```
SaaS    ─── 소프트웨어 제공 (최상위)
PaaS    ─── 개발 플랫폼 제공 (중간)
IaaS    ─── 인프라 제공 (최하위)
```

> [!tip] 기억법
> I(Infrastructure) → P(Platform) → S(Software) 순서로 추상화 수준 높아짐

## RAID (Redundant Array of Independent Disks)

| RAID | 방식 | 최소 디스크 | 가용 용량 | 특징 |
|------|------|-------------|-----------|------|
| **RAID 0** | 스트라이핑 | 2개 | 100% | 성능↑, 안전성 없음 |
| **RAID 1** | 미러링 | 2개 | 50% | 안전성↑, 용량 손실 |
| **RAID 5** | 패리티 분산 | **3개** | (n-1)/n | 성능+안전성 균형 |

```
RAID 0: [A1][A2]  [A3][A4]   ── 데이터를 분산 저장 (빠름)
         Disk1     Disk2

RAID 1: [A1][A1]  [A2][A2]   ── 동일 데이터 복제 (안전)
         Disk1     Disk2

RAID 5: [A1][A2][Ap] [B1][Bp][B2] [Cp][C1][C2]  ── 패리티 분산
         Disk1       Disk2         Disk3
```

> [!warning] RAID 5 최소 디스크
> RAID 5는 패리티 분산을 위해 **최소 3개** 디스크 필요 (2개 아님!)

## LVM (Logical Volume Manager)

```
물리 디스크 → PV(Physical Volume)
               ↓
           VG(Volume Group) ← 여러 PV 통합
               ↓
           LV(Logical Volume) ← 유연하게 분할
               ↓
           파일시스템 생성 + 마운트
```

| 용어 | 약어 | 설명 |
|------|------|------|
| Physical Volume | PV | 물리 디스크/파티션 |
| Volume Group | VG | PV들의 묶음 |
| Logical Volume | LV | VG에서 할당한 논리 볼륨 |

### LVM 관련 명령어
| 명령어 | 용도 |
|--------|------|
| `pvcreate` | PV 생성 |
| `vgcreate` | VG 생성 |
| `lvcreate` | LV 생성 |
| `pvdisplay` | PV 정보 확인 |
| `vgdisplay` | VG 정보 확인 |
| `lvdisplay` | LV 정보 확인 |
| `lvextend` | LV 크기 확장 |

> [!tip] LVM의 장점
> 파티션 크기를 **동적으로 변경** 가능. 기존 fdisk는 파티션 크기 변경이 어려움.
