# 네트워크 기초 (★★★)

#network #linux-master

## Overview Table (한눈에 비교)
| 항목 | TCP | UDP |
|------|-----|-----|
| 연결 | 연결지향 (3-way) | 비연결 |
| 신뢰성 | 높음 | 낮음 |
| 속도 | 느림 | 빠름 |
| 순서 보장 | O | X |
| 용도 | HTTP, FTP, SSH | DNS, 스트리밍 |

## 네트워크 토폴로지

| 토폴로지 | 구조 | 특징 |
|----------|------|------|
| **성형(Star)** | 중앙 허브 | 관리 용이, 허브 장애 시 전체 마비 |
| **망형(Mesh)** | 모든 노드 연결 | 안정성 최고, 비용 높음 |
| **버스형(Bus)** | 하나의 케이블 | 구축 간단, 충돌 문제 |
| **링형(Ring)** | 원형 연결 | 토큰 패싱, 한 노드 장애 시 전체 영향 |
| **트리형(Tree)** | 계층 구조 | 확장 용이, 상위 장애 시 하위 마비 |

## OSI 7계층 모델

```
7. 응용(Application)    ── HTTP, FTP, SMTP, DNS, Telnet
6. 표현(Presentation)   ── 암호화, 압축, 인코딩
5. 세션(Session)        ── 연결 설정/유지/종료
4. 전송(Transport)      ── TCP, UDP
3. 네트워크(Network)    ── IP, ICMP, ARP, RARP
2. 데이터링크(Data Link) ── 이더넷, MAC 주소
1. 물리(Physical)       ── 케이블, 허브, 리피터
```

### 계층별 주요 장비
| 계층 | 장비 |
|------|------|
| 1 (물리) | 허브, 리피터, 케이블 |
| 2 (데이터링크) | 스위치, 브리지 |
| 3 (네트워크) | 라우터 |
| 4+ | 게이트웨이 |

## TCP 3-way Handshaking

```
클라이언트           서버
    │── SYN ──────→│   1. 연결 요청
    │←─ SYN+ACK ──│   2. 요청 수락 + 확인
    │── ACK ──────→│   3. 연결 확립
```

## 주요 프로토콜

| 프로토콜 | 계층 | 용도 |
|----------|------|------|
| **IP** | 3 | 논리 주소 지정, 패킷 전달 |
| **ICMP** | 3 | 오류 보고, ping |
| **ARP** | 3 | IP → MAC 주소 변환 |
| **RARP** | 3 | MAC → IP 주소 변환 |
| **TCP** | 4 | 신뢰성 있는 연결 지향 전송 |
| **UDP** | 4 | 비연결, 빠른 전송 |
| **HTTP** | 7 | 웹 페이지 전송 (포트 80) |
| **HTTPS** | 7 | 암호화 웹 (포트 443) |
| **FTP** | 7 | 파일 전송 (포트 20/21) |
| **SSH** | 7 | 보안 원격 접속 (포트 22) |
| **Telnet** | 7 | 원격 접속 (포트 23, 비암호화) |
| **SMTP** | 7 | 메일 발송 (포트 25) |
| **POP3** | 7 | 메일 수신 (포트 110) |
| **IMAP** | 7 | 메일 수신, 서버 보관 (포트 143) |
| **DNS** | 7 | 도메인↔IP 변환 (포트 53) |

> [!warning] ARP vs RARP
> - **ARP**: IP 주소로 MAC 주소 알아냄 (주로 사용)
> - **RARP**: MAC 주소로 IP 주소 알아냄 (거의 사용 안 함)
