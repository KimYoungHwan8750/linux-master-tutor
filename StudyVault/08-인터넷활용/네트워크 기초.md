## 1. 통신망 종류

### 통신망 비교표

| 구분 | 범위 | 특징 | 기술/장비 |
|------|------|------|-----------|
| **LAN** (Local Area Network) | 빌딩 내/근접 거리 | **고속**, 자체 구축 | Ethernet, Token Ring, FDDI, MAC주소, UTP/BNC, 허브/스위치 |
| **MAN** (Metropolitan Area Network) | **도시** 단위 | 여러 LAN 연결 | DQDB, Metro Ethernet |
| **WAN** (Wide Area Network) | **국가/대륙** | 넓은 지역 | 전용선, VPN, MPLS |
| **SAN** (Storage Area Network) | 데이터센터 내 | **스토리지 전용** 고속 네트워크 | 파이버 채널, iSCSI |

> [!tip] LAN 핵심 기술
> - **Ethernet**: 가장 보편적인 LAN 기술 (IEEE 802.3)
> - **Token Ring**: 토큰을 돌려가며 통신 (IEEE 802.5)
> - **FDDI** (Fiber Distributed Data Interface): 광섬유 기반 링 구조
> - 케이블: **UTP** (비차폐 연선), **BNC** (동축 케이블)
> - 장비: **허브** (더미), **스위치** (MAC 주소 기반 전달)

## 2. LAN 토폴로지 (Topology)

### 토폴로지 비교표

| 토폴로지 | 구조 | 장점 | 단점 |
|----------|------|------|------|
| **성형 (Star)** | 중앙 허브/스위치에 모두 연결 | 확장 용이, 장애 격리 | **중앙 장치 고장 시 전체 마비** |
| **망형 (Mesh)** | 모든 장치가 일대일 연결 | **신뢰성 최고**, 장애 우회 | **비용 높음**, 관리 복잡 |
| **버스형 (Bus)** | 하나의 회선에 모두 연결 | **비용 저렴**, 설치 간단 | **병목 현상**, 장애 추적 어려움 |
| **링형 (Ring)** | 좌우 인접 장치와 연결 (원형) | 고속 전송, 충돌 없음 | **노드 장애 시 전체 영향** |
| **트리형 (Tree)** | 계층 구조 | 추가/삭제 용이, 관리 편리 | **병목 가능**, 상위 장애 시 하위 영향 |

```
성형 (Star)          망형 (Mesh)          버스형 (Bus)
    ┌─A               A───B               ═══A══B══C══D═══
    │                  │╲ ╱│
  ┌─HUB─┐             │ X │
  │  │   │             │╱ ╲│
  B  C   D             C───D

링형 (Ring)          트리형 (Tree)
  A─────B                  R
  │     │                ╱   ╲
  D─────C              A       B
                      ╱ ╲    ╱ ╲
                     C   D  E   F
```

## 3. OSI 7계층 모델

> [!info] OSI (Open Systems Interconnection) 모델은 **ISO**에서 개발한 네트워크 통신의 표준 참조 모델이다.

### OSI 7계층 상세 비교표

| 계층 | 이름 | 데이터 단위 | 주요 프로토콜/기능 | 주요 장비 |
|------|------|-------------|-------------------|-----------|
| **7** | **응용 (Application)** | Data | HTTP, FTP, SMTP, DNS, Telnet, SSH, SNMP | 게이트웨이, 프록시 |
| **6** | **표현 (Presentation)** | Data | 부호화, 압축, **암호화** (JPEG, MPEG, ASCII, SSL) | — |
| **5** | **세션 (Session)** | Data(Socket) | 연결 **설정/유지/해제**, 동기점 관리 | — |
| **4** | **전송 (Transport)** | **Segment** | **TCP**, **UDP**, 포트 번호 | 게이트웨이 |
| **3** | **네트워크 (Network)** | **Packet** | **IP**, ICMP, ARP, RARP, IGMP | **라우터** |
| **2** | **데이터링크 (Data Link)** | **Frame** | Ethernet, PPP, MAC 주소 | **스위치**, 브릿지 |
| **1** | **물리 (Physical)** | **Bit** | 전기/광 신호, 물리적 연결 | **리피터**, **허브**, 케이블 |

```
┌───────────────────────────────────────────────────┐
│  7. 응용 계층      │ HTTP, FTP, SMTP, DNS         │
├───────────────────────────────────────────────────┤
│  6. 표현 계층      │ 암호화, 압축, 부호화          │
├───────────────────────────────────────────────────┤
│  5. 세션 계층      │ 연결 설정/유지/해제           │
├───────────────────────────────────────────────────┤
│  4. 전송 계층      │ TCP, UDP  [Segment]          │
├───────────────────────────────────────────────────┤
│  3. 네트워크 계층  │ IP, ICMP, ARP  [Packet]      │
├───────────────────────────────────────────────────┤
│  2. 데이터링크     │ Ethernet, MAC  [Frame]       │
├───────────────────────────────────────────────────┤
│  1. 물리 계층      │ 전기 신호, 케이블  [Bit]     │
└───────────────────────────────────────────────────┘
```

> [!tip] OSI 계층 기억법 (아래→위)
> "**물**리 → **데**이터 → **네**트워크 → **전**송 → **세**션 → **표**현 → **응**용"
> → **"물데네전세표응"** 또는 역순 **"응표세전네데물"**

### 계층별 데이터 단위 정리

| 계층 | 데이터 단위 |
|------|-------------|
| 7~5 계층 | **Data** (메시지) |
| 4 전송 | **Segment** |
| 3 네트워크 | **Packet** |
| 2 데이터링크 | **Frame** |
| 1 물리 | **Bit** |

### 계층별 주요 장비

| 장비 | 계층 | 기능 |
|------|------|------|
| **리피터 (Repeater)** | 1 물리 | 신호 증폭/재생 |
| **허브 (Hub)** | 1 물리 | 다중 포트 리피터 (브로드캐스트) |
| **브릿지 (Bridge)** | 2 데이터링크 | MAC 주소 기반 프레임 전달 |
| **스위치 (Switch)** | 2 데이터링크 | 다중 포트 브릿지 (MAC 주소 학습) |
| **라우터 (Router)** | 3 네트워크 | IP 주소 기반 패킷 경로 설정 |
| **게이트웨이 (Gateway)** | 4~7 | 프로토콜 변환 |

## 4. 인터넷 프로토콜 상세

### 네트워크 계층 프로토콜

| 프로토콜 | 정식 이름 | 기능 |
|----------|-----------|------|
| **IP** | Internet Protocol | 정보를 주고받는 기본 **규약**, 비연결 |
| **ICMP** | Internet Control Message Protocol | IP 전달 오류 보고 **메시지** (ping 사용) |
| **IGMP** | Internet Group Management Protocol | **멀티캐스팅** 그룹 제어/관리 |
| **ARP** | Address Resolution Protocol | **IP → MAC** 주소 변환 |
| **RARP** | Reverse ARP | **MAC → IP** 주소 변환 |

> [!warning] ARP vs RARP 시험 빈출
> - **ARP**: IP 주소를 알고 MAC 주소를 모를 때 → **IP → MAC**
> - **RARP**: MAC 주소를 알고 IP 주소를 모를 때 → **MAC → IP**

## 5. TCP vs UDP

### 상세 비교표

| 구분 | TCP | UDP |
|------|-----|-----|
| 정식 이름 | Transmission Control Protocol | User Datagram Protocol |
| 연결 방식 | **연결 지향** (Connection-oriented) | **비연결** (Connectionless) |
| 신뢰성 | **높음** (데이터 전달 보장) | **낮음** (전달 보장 안 함) |
| 순서 보장 | O (순서대로 전달) | X (순서 보장 안 함) |
| 속도 | 느림 | **빠름** |
| 오류 제어 | O | X |
| 흐름 제어 | O | X |
| 데이터 단위 | Segment | Datagram |
| 대표 서비스 | FTP, Telnet, SMTP, **HTTP**, POP3 | **DNS**, DHCP, SNMP, TFTP |
| 헤더 크기 | 20바이트 | 8바이트 |

### TCP 3-Way Handshake (연결 설정)

```
  클라이언트                    서버
     │                          │
     │  ① SYN (연결 요청)  ──→  │
     │                          │
     │  ← ② SYN+ACK (응답)     │
     │                          │
     │  ③ ACK (확인)       ──→  │
     │                          │
     │     ═══ 연결 설정 ═══     │
```

| 단계 | 방향 | 플래그 | 설명 |
|------|------|--------|------|
| 1 | 클라이언트 → 서버 | **SYN** | 연결 요청 |
| 2 | 서버 → 클라이언트 | **SYN + ACK** | 연결 수락 + 확인 |
| 3 | 클라이언트 → 서버 | **ACK** | 연결 확인 |

### TCP 4-Way Handshake (연결 해제)

```
  클라이언트                    서버
     │                          │
     │  ① FIN (종료 요청)  ──→  │
     │                          │
     │  ← ② ACK (확인)         │
     │                          │
     │  ← ③ FIN (종료 요청)    │
     │                          │
     │  ④ ACK (확인)       ──→  │
     │                          │
     │     ═══ 연결 해제 ═══     │
```

| 단계 | 방향 | 플래그 | 설명 |
|------|------|--------|------|
| 1 | 클라이언트 → 서버 | **FIN** | 연결 종료 요청 |
| 2 | 서버 → 클라이언트 | **ACK** | 종료 확인 |
| 3 | 서버 → 클라이언트 | **FIN** | 서버도 종료 요청 |
| 4 | 클라이언트 → 서버 | **ACK** | 종료 최종 확인 |

### TCP 상태

| 상태 | 설명 |
|------|------|
| **LISTEN** | 서버가 연결 요청 **대기** 중 |
| **SYN_SENT** | 클라이언트가 SYN을 보내고 **응답 대기** |
| **SYN_RECEIVED** | 서버가 SYN을 받고 SYN+ACK 보낸 후 **ACK 대기** |
| **ESTABLISHED** | 연결이 **설정 완료** (데이터 전송 가능) |
| **CLOSE_WAIT** | 상대방의 FIN을 받고 **종료 대기** |
| **TIME_WAIT** | 연결 종료 후 일정 시간 **대기** (패킷 소멸 대기) |
| **FIN_WAIT_1** | FIN을 보낸 후 ACK **대기** |
| **FIN_WAIT_2** | ACK를 받고 상대방의 FIN **대기** |
| **CLOSING** | 양쪽 동시 FIN 전송 시 |
| **LAST_ACK** | FIN 보낸 후 마지막 ACK **대기** |
| **CLOSED** | 연결 **완전 종료** |

## 6. IPv4 클래스

### IPv4 클래스 분류표

| 클래스 | 첫 옥텟 범위 | 시작 비트 | 네트워크/호스트 | 용도 | 기본 서브넷 마스크 |
|--------|-------------|-----------|----------------|------|-------------------|
| **A** | **1~126** | 0 | 8비트/24비트 | **대규모** 네트워크 | 255.0.0.0 |
| **B** | **128~191** | 10 | 16비트/16비트 | **중규모** 네트워크 | 255.255.0.0 |
| **C** | **192~223** | 110 | 24비트/8비트 | **소규모** 네트워크 | 255.255.255.0 |
| **D** | **224~239** | 1110 | — | **멀티캐스트** | — |
| **E** | **240~255** | 1111 | — | **예약** (실험용) | — |

> [!warning] 주의할 IP 대역
> - **0.x.x.x**: 예약 (네트워크 자체를 표시)
> - **127.x.x.x**: **루프백 (Loopback)** 주소 (자기 자신, localhost)
> - **10.x.x.x**: 사설 IP (A클래스)
> - **172.16~31.x.x**: 사설 IP (B클래스)
> - **192.168.x.x**: 사설 IP (C클래스)

### 특수 IP 주소

| 주소 | 용도 |
|------|------|
| `127.0.0.1` | 루프백 (localhost) |
| `0.0.0.0` | 모든 네트워크 (기본 경로) |
| `255.255.255.255` | 브로드캐스트 |
| `10.0.0.0/8` | A클래스 사설 IP |
| `172.16.0.0/12` | B클래스 사설 IP |
| `192.168.0.0/16` | C클래스 사설 IP |

## 7. 서브넷 마스크

### 주소 체계 비교

| 항목 | 크기 | 표기법 |
|------|------|--------|
| **IPv4** | **32bit** | 10진수 4옥텟 (예: 192.168.1.1) |
| **IPv6** | **128bit** | 16진수 8그룹 (예: 2001:0db8::1) |
| **MAC** | **48bit** | 16진수 6옥텟 (예: 00:1A:2B:3C:4D:5E) |

### 서브넷 마스크 개념

- **서브넷 마스크**: 네트워크 부분과 호스트 부분을 구분하는 32비트 값
- 네트워크 비트 = 1, 호스트 비트 = 0

### 서브넷 계산 방법

```
예: IP 192.168.1.130, 서브넷 255.255.255.192 (/26)

서브넷 255.255.255.192 → 2진수: 11111111.11111111.11111111.11000000
                                                          ↑ 호스트 6비트

호스트 비트 수 = 6
서브넷당 호스트 수 = 2^6 - 2 = 62개 (네트워크/브로드캐스트 제외)
서브넷 수 = 2^2 = 4개

서브넷 범위:
  192.168.1.0   ~ 192.168.1.63   (첫 번째 서브넷)
  192.168.1.64  ~ 192.168.1.127  (두 번째 서브넷)
  192.168.1.128 ~ 192.168.1.191  (세 번째 서브넷) ← 130은 여기
  192.168.1.192 ~ 192.168.1.255  (네 번째 서브넷)
```

### CIDR 표기법

| CIDR | 서브넷 마스크 | 호스트 수 |
|------|-------------|-----------|
| /8 | 255.0.0.0 | 16,777,214 |
| /16 | 255.255.0.0 | 65,534 |
| /24 | 255.255.255.0 | 254 |
| /25 | 255.255.255.128 | 126 |
| /26 | 255.255.255.192 | 62 |
| /27 | 255.255.255.224 | 30 |
| /28 | 255.255.255.240 | 14 |
| /30 | 255.255.255.252 | 2 |
