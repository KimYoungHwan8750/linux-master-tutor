# 네트워크 명령어와 서비스 (★★★)

#network #linux-master

## Overview Table (한눈에 비교)
| 명령어 | 용도 |
|--------|------|
| `ifconfig` | 네트워크 인터페이스 설정/확인 |
| `ping` | 네트워크 연결 확인 (ICMP) |
| `traceroute` | 경로 추적 |
| `netstat` | 네트워크 상태/연결 확인 |
| `route` | 라우팅 테이블 관리 |
| `nslookup` | DNS 조회 |

## 네트워크 명령어

### ifconfig
```bash
ifconfig                    # 전체 인터페이스 확인
ifconfig eth0               # 특정 인터페이스
ifconfig eth0 192.168.1.10  # IP 설정
ifconfig eth0 up/down       # 활성화/비활성화
```

> [!tip] ip 명령어
> 최신 리눅스에서는 `ifconfig` 대신 `ip` 명령어 사용 추세
> - `ip addr show` = `ifconfig`
> - `ip link set eth0 up` = `ifconfig eth0 up`

### ping
```bash
ping 192.168.1.1       # 연결 확인 (ICMP Echo Request)
ping -c 5 google.com   # 5회만 전송
```

### traceroute
```bash
traceroute google.com  # 목적지까지 경로의 각 라우터 추적
```

### netstat
```bash
netstat -an    # 모든 연결, 숫자로 표시
netstat -tlnp  # TCP 리스닝 포트 + PID
```
| 옵션 | 의미 |
|------|------|
| `-a` | 모든 연결 |
| `-t` | TCP만 |
| `-u` | UDP만 |
| `-l` | 리스닝 상태만 |
| `-n` | 숫자로 표시 |
| `-p` | PID/프로그램명 |

### route
```bash
route              # 라우팅 테이블 확인
route add default gw 192.168.1.1  # 기본 게이트웨이 추가
```

### nslookup / dig
```bash
nslookup google.com    # DNS 질의
dig google.com         # 상세 DNS 질의
```

## 주요 네트워크 설정 파일

| 파일 | 용도 |
|------|------|
| `/etc/hosts` | IP↔호스트명 정적 매핑 |
| `/etc/resolv.conf` | DNS 서버 주소 설정 |
| `/etc/hostname` | 시스템 호스트명 |
| `/etc/sysconfig/network-scripts/ifcfg-*` | 네트워크 인터페이스 설정 (RHEL) |
| `/etc/network/interfaces` | 네트워크 설정 (Debian) |

## 원격 접속 서비스

| 서비스 | 포트 | 암호화 | 특징 |
|--------|------|--------|------|
| **Telnet** | 23 | ✗ | 평문 전송, 보안 취약 |
| **SSH** | 22 | ✓ | 암호화 통신, 현재 표준 |

> [!important] Telnet vs SSH
> Telnet은 **보안 취약**(평문 전송)으로 현재는 SSH 사용이 표준.
> 시험에서 "보안" + "원격 접속"이 키워드면 → SSH

### SSH 관련 명령어
```bash
ssh user@host           # 원격 접속
ssh -p 2222 user@host   # 포트 지정
scp file user@host:path # 파일 복사
sftp user@host          # FTP over SSH
```
