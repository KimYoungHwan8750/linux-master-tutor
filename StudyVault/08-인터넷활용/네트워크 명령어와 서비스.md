# 네트워크 명령어와 서비스 (★★★)

#linux-master #network #network-cmd #internet-svc

## 한눈에 비교

### 주요 포트 번호 총정리

| 서비스 | 포트 | 프로토콜 | 설명 |
|--------|------|----------|------|
| **FTP** | **21** | TCP | 파일 전송 |
| **SSH/SFTP** | **22** | TCP | 보안 셸/보안 파일 전송 |
| **Telnet** | **23** | TCP | 원격 접속 (비암호화) |
| **SMTP** | **25** | TCP | 메일 **송신** |
| **DNS** | **53** | TCP/UDP | 도메인→IP 변환 |
| **HTTP** | **80** | TCP | 웹 서비스 |
| **POP3** | **110** | TCP | 메일 **수신** (다운로드) |
| **IMAP** | **143** | TCP | 메일 **수신** (서버 보관) |
| **HTTPS** | **443** | TCP | 보안 웹 서비스 |
| **SMTPS** | **465** | TCP | 보안 메일 송신 |

## 1. 네트워크 명령어

### 1-1. ifconfig (interface configuration)

> 네트워크 인터페이스 **설정 및 확인** (레거시, ip 명령으로 대체 추세)

| 명령어 | 설명 |
|--------|------|
| `ifconfig` | 모든 활성 인터페이스 정보 확인 |
| `ifconfig -a` | **모든** 인터페이스 (비활성 포함) 확인 |
| `ifconfig eth0` | eth0 인터페이스 정보 확인 |
| `ifconfig eth0 up` | eth0 인터페이스 **활성화** |
| `ifconfig eth0 down` | eth0 인터페이스 **비활성화** |
| `ifconfig eth0 192.168.1.10` | eth0에 IP 주소 설정 |
| `ifconfig eth0 192.168.1.10 netmask 255.255.255.0` | IP와 서브넷 마스크 설정 |
| `ifconfig eth0 hw ether 00:11:22:33:44:55` | MAC 주소 변경 |

### 1-2. ip (iproute2)

> ifconfig를 **대체**하는 현대적 네트워크 설정 명령어

| 명령어 | 설명 | ifconfig 대응 |
|--------|------|--------------|
| `ip addr` (또는 `ip a`) | IP 주소 정보 확인 | `ifconfig` |
| `ip addr show eth0` | 특정 인터페이스 정보 | `ifconfig eth0` |
| `ip addr add 192.168.1.10/24 dev eth0` | IP 주소 추가 | `ifconfig eth0 IP` |
| `ip addr del 192.168.1.10/24 dev eth0` | IP 주소 삭제 | — |
| `ip link` | 링크 상태 확인 | — |
| `ip link set eth0 up` | 인터페이스 활성화 | `ifconfig eth0 up` |
| `ip link set eth0 down` | 인터페이스 비활성화 | `ifconfig eth0 down` |
| `ip route` | 라우팅 테이블 확인 | `route` |
| `ip route add default via 192.168.1.1` | 기본 게이트웨이 설정 | `route add default gw` |
| `ip neigh` | ARP 테이블 확인 | `arp` |

### 1-3. nslookup

> **DNS** 질의를 통해 도메인→IP 또는 IP→도메인을 조회

```bash
nslookup www.example.com          # 도메인의 IP 조회
nslookup 8.8.8.8                  # 역방향 조회
nslookup www.example.com 8.8.8.8  # 특정 DNS 서버로 질의
```

### 1-4. ping

> **ICMP** 프로토콜을 사용하여 네트워크 **통신 여부** 점검

| 옵션 | 설명 | 예시 |
|------|------|------|
| `-c count` | 전송할 패킷 **횟수** 지정 | `ping -c 4 192.168.1.1` |
| `-i interval` | 패킷 전송 **간격**(초) | `ping -i 2 192.168.1.1` |
| `-s size` | 패킷 **크기**(바이트) | `ping -s 1024 192.168.1.1` |
| `-t ttl` | **TTL** 값 설정 | `ping -t 64 192.168.1.1` |
| `-w timeout` | 전체 **제한시간**(초) | `ping -w 10 192.168.1.1` |

### 1-5. traceroute

> 목적지까지의 **패킷 경로(라우팅 경로)** 추적

```bash
traceroute www.example.com        # 경로 추적
traceroute -n www.example.com     # DNS 조회 없이 IP만 표시
traceroute -m 20 www.example.com  # 최대 홉 수 지정
```

- Windows에서는 `tracert` 명령어 사용

### 1-6. netstat (network statistics)

> 네트워크 연결 **상태**, 라우팅 테이블, 인터페이스 통계 확인

| 옵션 | 설명 |
|------|------|
| `-a` | **모든** 연결과 대기 포트 표시 (all) |
| `-n` | 주소/포트를 **숫자**로 표시 (numeric) |
| `-t` | **TCP** 연결만 표시 |
| `-u` | **UDP** 연결만 표시 |
| `-l` | **LISTEN** 상태만 표시 (대기 중 포트) |
| `-p` | **프로세스** PID/이름 표시 |
| `-r` | **라우팅** 테이블 표시 |
| `-s` | 프로토콜별 **통계** |
| `-i` | **인터페이스** 통계 |

```bash
# 자주 쓰는 조합
netstat -an            # 모든 연결을 숫자로 표시
netstat -tlnp          # TCP 대기 포트 + PID
netstat -rn            # 라우팅 테이블 (숫자)
netstat -s             # 프로토콜별 통계
```

### 1-7. ss (socket statistics)

> netstat를 **대체**하는 현대적 소켓 통계 명령어 (더 빠르고 상세)

| 명령어 | 설명 |
|--------|------|
| `ss -a` | 모든 소켓 표시 |
| `ss -t` | TCP 소켓만 |
| `ss -u` | UDP 소켓만 |
| `ss -l` | LISTEN 상태만 |
| `ss -n` | 숫자 표시 |
| `ss -p` | 프로세스 정보 |
| `ss -tlnp` | TCP 대기 포트 + PID |

### 1-8. route

> **라우팅 테이블** 확인 및 관리

| 명령어 | 설명 |
|--------|------|
| `route` | 라우팅 테이블 확인 |
| `route -n` | 숫자로 표시 (DNS 조회 안 함) |
| `route add default gw 192.168.1.1` | 기본 **게이트웨이** 추가 |
| `route del default` | 기본 게이트웨이 삭제 |
| `route add -net 10.0.0.0 netmask 255.0.0.0 gw 192.168.1.1` | 네트워크 라우트 추가 |

### 1-9. ethtool

> 네트워크 인터페이스의 **물리적 연결 상태** 확인

```bash
ethtool eth0           # eth0의 속도, 듀플렉스, 링크 상태 확인
ethtool -i eth0        # 드라이버 정보
ethtool -S eth0        # 통계 정보
```

### 1-10. mii-tool

> 네트워크 인터페이스의 **속도/전송 모드** 확인

```bash
mii-tool eth0          # 속도, 전이중/반이중 확인
mii-tool -v eth0       # 상세 정보
```

### 1-11. arp

> **ARP 테이블** 확인, 추가, 삭제 (IP↔MAC 매핑)

| 명령어 | 설명 |
|--------|------|
| `arp` | ARP 테이블 확인 |
| `arp -a` | 모든 항목 표시 |
| `arp -n` | 숫자로 표시 |
| `arp -s IP MAC` | 정적 ARP 항목 **추가** |
| `arp -d IP` | ARP 항목 **삭제** |

```bash
arp -s 192.168.1.100 00:11:22:33:44:55   # 정적 추가
arp -d 192.168.1.100                      # 삭제
```

### 네트워크 명령어 레거시 vs 현대 비교

| 기능 | 레거시 | 현대적 대체 |
|------|--------|-------------|
| IP/인터페이스 설정 | `ifconfig` | **`ip addr`** / **`ip link`** |
| 라우팅 테이블 | `route` | **`ip route`** |
| 네트워크 통계 | `netstat` | **`ss`** |
| ARP 테이블 | `arp` | **`ip neigh`** |

## 2. 인터넷 서비스

### 2-1. WWW (World Wide Web)

- **HTTP** 프로토콜 사용 (포트 80)
- **멀티미디어** 및 **하이퍼텍스트** 기반
- 웹 브라우저를 통한 정보 탐색
- HTTPS (포트 443): SSL/TLS 암호화

### 2-2. Gopher

- **미네소타 대학**에서 개발
- **터미널(텍스트) 기반** 검색 시스템
- 메뉴 구조로 정보 탐색
- WWW 이전의 인터넷 서비스 (현재 거의 미사용)

### 2-3. 전자 메일

| 프로토콜 | 포트 | 기능 | 방향 |
|----------|------|------|------|
| **SMTP** | **25** | 메일 **송신** (Simple Mail Transfer Protocol) | 보내기 |
| **SMTPS** | **465** | SMTP + **SSL/TLS** 보안 | 보내기 (보안) |
| **POP3** | **110** | 메일 **수신** (다운로드 후 서버에서 삭제) | 받기 |
| **IMAP** | **143** | 메일 **수신** (서버에 보관, 동기화) | 받기 |

> [!tip] POP3 vs IMAP
> - **POP3**: 메일을 **다운로드**하고 서버에서 삭제 (오프라인 열람)
> - **IMAP**: 메일을 서버에 **보관**하고 동기화 (여러 기기 접근)

### 2-4. FTP (File Transfer Protocol)

- **파일 전송** 프로토콜
- 포트: **21** (제어 연결), 20 (데이터 연결)
- **SFTP**: SSH 기반 보안 파일 전송 (포트 **22**)
- **FTPS**: FTP + SSL/TLS

### 2-5. Telnet

- **원격 접속** 프로토콜 (포트 **23**)
- 데이터가 **암호화되지 않음** (보안 취약)
- SSH로 대체됨

| 옵션 | 설명 |
|------|------|
| `-l user` | 사용자명 지정 |
| `-a` | 자동 로그인 |
| `-d` | 디버그 모드 |

```bash
telnet 192.168.1.100           # 기본 접속
telnet -l admin 192.168.1.100  # 사용자 지정 접속
telnet 192.168.1.100 80        # 특정 포트 접속
```

### 2-6. SSH (Secure Shell)

- **보안 셸** (포트 **22**)
- 데이터 **암호화** 전송
- Telnet의 보안 대체

| 옵션 | 설명 |
|------|------|
| `-p 포트` | **포트** 번호 지정 |
| `-l 사용자` | **로그인** 사용자 지정 |
| `-i 키파일` | **인증 키** 파일 지정 |
| `-X` | X11 포워딩 활성화 |

```bash
ssh 192.168.1.100              # 기본 접속
ssh -p 2222 192.168.1.100      # 포트 지정
ssh -l admin 192.168.1.100     # 사용자 지정
ssh admin@192.168.1.100        # 사용자@호스트
```

### 2-7. IRC (Internet Relay Chat)

- **실시간 채팅** 프로토콜
- **클라이언트-서버** 구조
- 채널(#) 기반 그룹 대화

### 2-8. SAMBA

- **Windows-Linux** 간 **파일/프린터 공유** 서비스
- **SMB/CIFS** 프로토콜 사용
- Windows 네트워크 환경에서 리눅스 자원 공유 가능
- 설정 파일: `/etc/samba/smb.conf`

### 2-9. DNS (Domain Name System)

- **도메인 이름** → **IP 주소** 변환 서비스
- 포트: **53** (TCP/UDP)
- 설정 파일: `/etc/resolv.conf` (DNS 서버 주소)

## 3. 주요 설정 파일

| 파일 | 역할 | 예시 내용 |
|------|------|-----------|
| `/etc/hosts` | IP와 **호스트명** 직접 매핑 | `192.168.1.100  myserver` |
| `/etc/resolv.conf` | **DNS 서버** 주소 지정 | `nameserver 8.8.8.8` |
| `/etc/sysconfig/network` | **네트워크** 기본 설정 (Red Hat 계열) | `HOSTNAME=myserver` |
| `/etc/sysconfig/network-scripts/ifcfg-eth0` | **인터페이스** 설정 (Red Hat 계열) | `IPADDR=192.168.1.10` |
| `/etc/hostname` | **호스트명** 설정 (systemd) | `myserver` |
| `/etc/services` | **서비스-포트** 매핑 | `ssh  22/tcp` |

```
# /etc/resolv.conf 예시
nameserver 8.8.8.8
nameserver 8.8.4.4
search example.com

# /etc/hosts 예시
127.0.0.1    localhost
192.168.1.10 webserver
192.168.1.20 dbserver
```
