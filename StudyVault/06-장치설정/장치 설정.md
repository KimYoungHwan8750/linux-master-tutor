## 1. 프린터 시스템

### 1-1. LPRng (Line Printer next generation)

- **BSD 계열** 인쇄 시스템
- `/etc/printcap` 파일로 프린터 설정
- LPD (Line Printer Daemon) 프로토콜 사용
- 기존 BSD 인쇄 시스템(lpd)의 개선 버전
- **lpr**, **lpq**, **lprm** 등 BSD 계열 명령어 사용

### 1-2. CUPS (Common Unix Printing System)

- **애플(Apple)**에서 개발한 인쇄 시스템
- 사용 프로토콜: **HTTP**, **IPP**(Internet Printing Protocol), **SMB**
- **웹 브라우저**를 통한 프린터 관리 가능 (포트 **631**)
  - 접속 주소: `http://localhost:631`
- 현재 리눅스 표준 인쇄 시스템

#### CUPS 설정 파일 (/etc/cups/)

| 파일 | 설명 |
|------|------|
| `/etc/cups/cupsd.conf` | CUPS **데몬(서버)** 설정 파일 (포트, 접근제어, 로그 등) |
| `/etc/cups/printers.conf` | **프린터** 설정 정보 (프린터별 장치, 드라이버 등) |
| `/etc/cups/classes.conf` | 프린터 **클래스(그룹)** 설정 |
| `/etc/cups/ppd/` | PPD (PostScript Printer Description) 파일 디렉터리 |

> [!tip] CUPS 키워드 정리
> - 애플 개발
> - HTTP/IPP 프로토콜
> - 포트 631
> - 웹 기반 관리
> - `/etc/cups/cupsd.conf`

### 1-3. 프린터 명령어

#### BSD 계열 명령어

| 명령어 | 기능 | 주요 옵션 | 예시 |
|--------|------|-----------|------|
| `lpr` | 프린터에 **출력 요청** | `-P 프린터명`, `-# 매수` | `lpr -P lp0 file.txt` |
| `lpq` | 프린터 **큐(대기열) 목록** 확인 | `-P 프린터명`, `-l`(상세) | `lpq -P lp0` |
| `lprm` | 프린터 큐에서 **작업 삭제** | `-P 프린터명`, `작업번호` | `lprm -P lp0 123` |
| `lpc` | 프린터 **상태 확인** 및 제어 | `status`, `start`, `stop` | `lpc status all` |

#### System V 계열 명령어

| 명령어 | 기능 | 주요 옵션 | 예시 |
|--------|------|-----------|------|
| `lp` | 프린터에 **출력 요청** | `-d 프린터명`, `-n 매수` | `lp -d lp0 file.txt` |
| `lpstat` | 프린터 **큐 상태** 확인 | `-a`(수락상태), `-p`(프린터상태), `-t`(전체정보) | `lpstat -t` |
| `cancel` | 프린터 **작업 취소** | `작업ID`, `-a 프린터명` | `cancel lp0-123` |

#### BSD vs System V 명령어 비교표

| 기능 | BSD 계열 | System V 계열 |
|------|----------|---------------|
| 출력 요청 | `lpr` | `lp` |
| 큐 확인 | `lpq` | `lpstat` |
| 작업 삭제/취소 | `lprm` | `cancel` |
| 상태 제어 | `lpc` | — |
| 프린터 지정 옵션 | `-P` | `-d` |
| 매수 지정 옵션 | `-#` | `-n` |

> [!warning] 시험 주의
> - **lpr**(BSD) vs **lp**(System V) — 출력 요청 명령어 구분
> - **lpq**(BSD) vs **lpstat**(System V) — 큐 상태 확인 명령어 구분
> - **lprm**(BSD) vs **cancel**(System V) — 작업 삭제 명령어 구분

### LPRng vs CUPS 비교표

| 구분 | LPRng | CUPS |
|------|-------|------|
| 계열 | BSD | 애플 개발 |
| 설정 파일 | `/etc/printcap` | `/etc/cups/cupsd.conf` |
| 프로토콜 | LPD | HTTP, IPP, SMB |
| 웹 관리 | X | O (포트 631) |
| 현재 상태 | 레거시 | 리눅스 표준 |

## 2. 사운드 카드

### 2-1. OSS (Open Sound System)

- **유닉스 계열** 운영체제에서 사운드를 만들고 캡처하는 인터페이스
- **POSIX** 표준 시스템 호출(read, write, ioctl) 기반
- 장치 파일: `/dev/dsp`, `/dev/mixer`, `/dev/audio`
- 현재는 ALSA에 의해 대체되었으나 호환 모듈 제공

### 2-2. ALSA (Advanced Linux Sound Architecture)

- **리눅스 커널** 요소로 포함된 사운드 시스템
- OSS를 대체하는 **현대적 표준**
- 특징:
  - 사운드 카드 **자동 구성** 지원
  - **다중 사운드 장치** 동시 관리 가능
  - **OSS 호환 모듈** 제공
  - 커널 모듈로 동작 (`snd-*`)

### 사운드 관련 명령어

| 명령어 | 설명 | 비고 |
|--------|------|------|
| `alsactl` | ALSA 사운드 카드 **제어** | 설정 저장/복원: `alsactl store`, `alsactl restore` |
| `alsamixer` | **커서(ncurses) 기반** 오디오 믹서 프로그램 | 터미널에서 볼륨 조절 |
| `cdparanoia` | CD에서 **오디오 트랙을 추출**(리핑) | CD → WAV 파일 변환 |

### OSS vs ALSA 비교표

| 구분 | OSS | ALSA |
|------|-----|------|
| 대상 OS | 유닉스 전반 | **리눅스 전용** |
| 기반 | POSIX 시스템 호출 | **커널 모듈** |
| 자동 구성 | X | O |
| 다중 장치 | 제한적 | O |
| 현재 상태 | 레거시 | **리눅스 표준** |
| 도구 | — | alsactl, alsamixer |

## 3. 스캐너

### 3-1. SANE (Scanner Access Now Easy)

- 이미지 관련 하드웨어를 제어하는 **API (Application Programming Interface)**
- 스캐너, 디지털 카메라 등 이미지 입력 장치 지원
- **백엔드(Backend)**: 하드웨어 드라이버 계층
- **프론트엔드(Frontend)**: 사용자 인터페이스 계층

```
┌──────────────────────────────────────┐
│  프론트엔드 (Frontend)               │
│  ┌──────┐  ┌──────┐  ┌──────────┐   │
│  │ XSANE│  │xcam  │  │scanimage │   │
│  └──┬───┘  └──┬───┘  └────┬─────┘   │
│     └─────────┴────────────┘         │
│              SANE API                │
│     ┌─────────┴────────────┐         │
│  ┌──┴───┐  ┌──┴───┐  ┌──┴──┐       │
│  │드라이버│  │드라이버│  │드라이버│    │
│  └──┬───┘  └──┬───┘  └──┬──┘       │
│     │         │         │           │
│  백엔드 (Backend) - 하드웨어 계층    │
│  ┌──┴───┐  ┌──┴───┐  ┌──┴──┐       │
│  │스캐너 │  │카메라 │  │기타  │      │
│  └──────┘  └──────┘  └─────┘       │
└──────────────────────────────────────┘
```

### 3-2. XSANE (X-based Scanner Access Now Easy)

- SANE의 **X-Windows 기반 GUI 프론트엔드**
- 그래픽 환경에서 스캔 작업 수행
- 기능: 미리보기, 해상도 설정, 파일 형식 선택, 이미지 편집

### 스캐너 관련 명령어

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `sane-find-scanner` | 시스템에 연결된 **스캐너 장치 검색** | `sane-find-scanner` |
| `scanimage` | 명령줄에서 **이미지 스캔** | `scanimage --format=png > scan.png` |
| `scanadf` | **ADF**(자동 문서 급지 장치)를 통한 **다중 페이지 스캔** | `scanadf -o scan%d.png` |
| `xcam` | **GUI 기반** 스캔 프로그램 (X-Windows) | `xcam` |

> [!tip] 스캐너 명령어 구분
> - `sane-find-scanner`: **찾기** (장치 검색)
> - `scanimage`: **CLI 스캔** (명령줄)
> - `scanadf`: **ADF 다중 스캔** (여러 장)
> - `xcam`: **GUI 스캔** (그래픽)
> - `XSANE`: SANE의 **X-Windows GUI 프론트엔드**
